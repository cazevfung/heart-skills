# 正式简洁（Official Formal）

## 一句话描述

简洁、专业、信息密度高，适合版本说明、维护公告、海外或偏硬核向游戏。

## 语气 / 口吻

- 正式、客观，不煽情
- 句子短而清晰，少用修辞
- 以「信息传达」为主：时间、范围、补偿、改动项一目了然
- 避免口语化、梗、过度拟人

## 称呼与署名

- 称呼玩家：Players / 各位玩家 / 玩家（中英文按 language 参数）
- 自称：The Team / Development Team / 开发团队 / 运营团队
- 落款：简短署名即可，如「— [Game Name] Team」「— 《XXX》运营团队」

## 用词偏好与禁忌

- **偏好**：将、于、进行、届时、完成、发放、更新、修复、调整
- **避免**：网络流行语、梗、过多感叹号、卖萌语气、「哈」「呀」「哦」等语气词堆砌
- **避免**：模糊表述（如「很快」「一会儿」），改用具体时间或版本号

## 结构建议

- **开头**：直接说明公告类型（Maintenance / Update / Event）或版本号。
- **正文**：分条或分节：时间（UTC/本地）、影响范围、补偿内容、变更列表；每条简短一句。
- **结尾**：一句感谢或「如有问题请联系客服/社区」，再落款。

## 公告类型差异

- **维护公告**：Title 含 Maintenance + 日期；正文：时间窗口、影响（登录/部分功能）、预计时长、补偿清单。
- **版本公告**：Version X.X.X 或 Patch Notes；正文：New / Changed / Fixed 分列，每条一行。
- **活动公告**：活动名称 + 时间 + 参与方式 + 奖励概要，不展开剧情或情感描述。

## 示例片段（脱敏）

- 开头示例：「Maintenance Notice — [Date]」/「【维护公告】X 月 X 日」
- 正文示例：「Duration: 02:00 – 06:00 UTC. All servers will be unavailable. Compensation: [list].」
- 中文示例：「维护时间：X 月 X 日 02:00–06:00（UTC）。维护期间无法登录。补偿内容：[列表]。感谢理解。—— 《XXX》运营团队」
