# 结构：标准版 (standard)

适合 3–6 分钟的版本更新解说视频，信息完整、层次清晰。

## 段落顺序与要求

| 顺序 | 段落 id | 说明 | 建议时长/篇幅 |
|------|---------|------|----------------|
| 1 | hook | 开场钩子：打招呼 + 本期主题（某游戏某版本更新） | 约 15–30 秒 / 50–80 字 |
| 2 | overview | 版本概览：版本号、发布日期、一句话概括本版重点（如「大版本」「平衡为主」「Bug 修复」） | 约 20–40 秒 / 60–120 字 |
| 3 | new_content | 新内容：新角色/新赛季/新活动等，按 patch 顺序或重要性排列 | 按实际内容，每项 1–3 句 |
| 4 | balance | 平衡性调整：英雄/装备/技能改动，可简要说明「加强/削弱/重做」 | 按实际内容，每条 1–2 句 |
| 5 | systems | 系统与功能优化：UI、匹配、商城、社交等非数值改动 | 按实际内容，可合并为一段 |
| 6 | bugfixes | Bug 修复：挑影响大的提 1–2 句，其余概括为「另有若干 Bug 修复」 | 约 20–40 秒 |
| 7 | outro | 总结 + CTA：本版亮点回顾 + 「详细 patch 见简介/评论区」「记得三连」等 | 约 15–30 秒 / 40–80 字 |

## 时间戳（可选）

若稿子需用于剪辑，可在每段开头加时间戳占位，格式为 `[MM:SS]`，由生成时按建议时长估算填写，例如：

- `[0:00]` hook  
- `[0:20]` overview  
- `[1:00]` new_content  
- …  
- `[4:30]` outro  

是否带时间戳可由用户在请求中说明（如「带时间戳」），或在本 skill 后续扩展为 structure 级别配置。

## 与风格的配合

- 各段落内容须同时满足当前选中的**风格**（如 formal 则全篇正式，casual 则全篇口语化）。
- 若 changelog 中某类内容缺失（如无新角色），对应段落可写「本版本无新角色/无平衡调整」等一句带过，保持结构完整。
