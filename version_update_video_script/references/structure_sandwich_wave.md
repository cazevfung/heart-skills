# 结构：三明治波浪式 (sandwich_wave)

适合 15–25 分钟（完整版）或 5–8 分钟（精简版）的版本更新解说视频。

源自《动物森友会》Nintendo Direct 10.15.2021 的经典结构，适用于生活模拟、治愈系、轻社交类游戏的大版本更新发布。风格特征：导游式口吻、情绪波浪、画面留白、生活感沉浸。

---

## 核心结构：三明治五段式

```
[开场钩子] → [免费内容堆叠] → [情感转折/悬念] → [付费内容深度展示] → [收尾号召]
```

### 第一段：开场钩子 (Hook) — 0:00-0:36 范例

**功能**：建立期待，给出时间锚点，设定情绪基调

**结构要素**：
| 元素 | 说明 | 时长建议 |
|------|------|----------|
| 品牌标识音 | 短暂的品牌声音（如 Nintendo Switch Snap） | 2-3 秒 |
| 环境音铺垫 | 海浪声、风声等，建立场景感 | 3-5 秒 |
| 主题音乐渐入 | 平静的原声吉他/钢琴，设定"治愈"基调 | 5-8 秒 |
| 旁白开场 | 直接问候 + 本期主题 + 发布时间 | 15-25 秒 |

**台词技巧**：
- 第二人称直接对话："Hello, everyone! Thank you for tuning in..."
- 一句话说清三件事：我是谁 + 今天讲什么 + 什么时候发布
- 示例："Today, we'll be going over the details of the update... The update is scheduled to be released on November 5th."

**情绪曲线**：平静 → 轻微期待

### 第二段：免费内容堆叠 (Free Content Stack) — 0:36-12:12 范例

**功能**：持续制造惊喜，建立"欠人情"心理

**核心原则：波浪式节奏**

不要平铺直叙，要在"兴奋"和"平静"之间交替：

```
[平静介绍] → [音乐突变/悬念] → [功能揭晓] → [角色互动/Animalese] → [细节补充] → [音乐淡出/平静]
```

#### 2.1 单个功能模块的标准节奏（以咖啡厅为例）

**Step 1: 悬念钩子（4-6 秒）**
- 音乐突变（funky acoustic music）
- 旁白短句："Something's coming to the museum!"
- 音乐淡出，给画面留空

**Step 2: 功能揭晓（10-15 秒）**
- 新音乐渐入（gentle lounge music）
- 旁白介绍："This is the Roost, a café, owned by the composed Brewster."
- 句式：This is [名称], a [类型], [定语从句描述]

**Step 3: 角色互动/Animalese（5-10 秒）**
- 角色说话声（Animalese），无旁白
- 功能：情感填充、节奏打断、制造"生活感"
- 旁白在此时必须完全沉默

**Step 4: 细节补充（15-25 秒）**
- 旁白继续，介绍玩法细节
- 短句为主，逗号切割："Kick back and relax, while Brewster serves you up a cup of his finest, hand-selected coffee."
- 邀请式语气："Why not savor a relaxing cup of joe together?"

**Step 5: 音乐淡出（3-5 秒）**
- 给上一个功能收尾
- 为下一个功能的音乐突变做准备

#### 2.2 内容堆叠的密度控制

**完整版（15-25 分钟）**：
- 每个功能模块 45-90 秒
- 模块之间用音乐过渡
- 可以深入展示 8-15 个功能点

**精简版（5-8 分钟）**：
- 每个功能模块 20-30 秒
- 快速切换，制造"还有？"的惊喜感
- 展示 6-10 个功能点

#### 2.3 功能介绍的句式模板

**新地点/建筑**：
```
This is [名称], a [类型], [一句话描述].
[Animalese 留白]
You can [功能1], [功能2].
[Animalese 留白]
Why not [邀请式结尾]?
```

**新角色**：
```
This is [角色名]. [一句话定位].
[Animalese 留白]
[角色特点/功能].
[情感点评，如："It's worth the journey just to be serenaded by his sea shanties."]
```

**新系统/玩法**：
```
[系统名] is coming to [游戏]!
[一句话解释这是什么]
[具体例子："Combine ingredients like tomatoes, wheat, sugarcane..."]
[情感反应："Mmm, looks tasty!"]
[延伸玩法]
```

#### 2.4 音乐情绪对照表

| 音乐类型 | 适用场景 | 情绪功能 |
|----------|----------|----------|
| Tranquil acoustic | 开场、过渡、收尾 | 平静、治愈、建立基调 |
| Funky acoustic | 悬念钩子、功能揭晓前 | 轻微兴奋、引起注意 |
| Gentle lounge | 室内场景、社交功能 | 温馨、放松 |
| Peppy acoustic | 活跃功能（运动、收集） | 轻快、有趣 |
| Festive piano | 多人互动、庆祝场景 | 欢乐、社交感 |
| Chirpy acoustic | 技术功能、工具更新 | 清新、实用感 |
| Cheery synth | 新应用、科技功能 | 现代、便捷 |
| Peaceful acoustic | 探索、发现类功能 | 神秘、期待 |

### 第三段：情感转折/悬念 (The Twist) — 12:12-12:30 范例

**功能**：制造情绪低谷 → 立即反弹，为付费内容铺垫

**标准结构**：

**Step 1: 情感低点声明（10-15 秒）**
- 音乐保持平静
- 旁白："This will be the last major free content update for [游戏名]."
- 不要解释原因，让失落感自然产生

**Step 2: 立即转折（3-5 秒）**
- 环境音打破平静（如飞机引擎声）
- 旁白："Oh? Something new is on the horizon!"
- 音乐突变（relaxing acoustic jazz）

**情绪曲线**：
```
平静 → 失落（低点）→ 好奇（转折）→ 期待（新高）
```

**关键技巧**：
- 转折必须快，不要让失落感停留超过 5 秒
- 用疑问句/感叹句制造悬念，不要直接说"还有付费内容"
- 环境音（引擎、门铃、电话声）比音乐更能打破情绪

### 第四段：付费内容深度展示 (Paid Content Deep Dive) — 12:30-22:37 范例

**功能**：从"展示功能"转向"展示生活方式"，促成购买决策

**与第二段的核心区别**：

| 维度 | 第二段（免费内容） | 第四段（付费内容） |
|------|-------------------|-------------------|
| 节奏 | 快节奏堆叠 | 慢节奏沉浸 |
| 展示方式 | 功能点罗列 | 完整流程演示 |
| 深度 | 点到为止（30-90秒/功能） | 深度展开（2-5分钟/功能） |
| 叙事逻辑 | "有什么" | "怎么玩" |
| 情感目标 | 惊喜 | 向往 |

#### 4.1 深度展示的标准流程

以《Happy Home Paradise》DLC 为例：

**Step 1: 场景建立（15-25 秒）**
- 展示新地图/区域
- 旁白："Our destination is this archipelago, made up of several islands."
- 一句话定位："Known as a resort, this locale is popular among characters searching for their ideal vacation homes."

**Step 2: 角色引入（10-15 秒）**
- 介绍核心 NPC
- 旁白："After a reference from Tom Nook, you'll assist Paradise Planning..."
- 说明玩家角色："Through your work, you'll be in charge of coordinating dream lifestyles for characters."

**Step 3: 完整流程演示（3-5 分钟）**
不要跳跃，要展示一个完整的任务流程：

```
接任务 → 选地点 → 布置过程 → 完成反馈
```

**具体节奏**：
1. **接任务**：展示 NPC 请求（带 Animalese）
2. **选地点**：展示选择界面，旁白解释不同选项
3. **布置过程**：
   - 放置家具（实时展示）
   - 旁白简短提示："Create depth by adding lighting to the ceiling."
   - 情感点评："It looks fantastic!"
4. **完成反馈**：展示角色反应（Animalese + 表情）

**Step 4: 进阶功能展开（2-3 分钟）**
在基础流程之后，展示深度玩法：
- "As you continue designing vacation homes, you'll acquire various design techniques."
- 逐个展示进阶功能（调整房间大小、隔断墙、灯光调节等）
- 每个功能配一句应用场景说明

**Step 5: 社交/分享功能（1-2 分钟）**
- 展示玩家可以如何分享/展示成果
- 强调社区属性："share your design with people from all over the world"

#### 4.2 深度展示的台词技巧

**过程性描述**：
```
"First, select the island... Each island has distinctive features. 
Pick the island that best matches the request, then, head on over to the site!"
```

**即时反馈**：
```
"Seems like the client is pretty happy!"
"It looks fantastic!"
```

**延伸想象**：
```
"Why not give your furniture a new flourish?"
"Who knows? Someday, after you've designed a certain number of homes, you might even..."
```

---

### 第五段：收尾号召 (Outro/CTA) — 22:37-22:57 范例

**功能**：情感收束，明确行动召唤

**标准结构**：

**Step 1: 感谢观看（5-8 秒）**
- 音乐回到开场基调（tranquil）
- 旁白："Thanks for watching the [游戏名] Direct!"

**Step 2: 情感升华（10-15 秒）**
- 不说功能，说感受
- "We hope you'll savor an even sweeter island life, and relish in the new design experience."
- 关键词：savor, relish, enjoy, experience

**Step 3: 发售日号召（5-8 秒）**
- 明确日期："See ya on November 5th!"
- 简短有力，不要加其他信息

**Step 4: 品牌音收尾（3-5 秒）**
- 环境音（海浪声）
- 音乐淡出

## 旁白-画面配合的留白艺术

### 什么时候闭嘴？

| 场景 | 留白时长 | 目的 |
|------|----------|------|
| 角色 Animalese | 3-10 秒 | 情感填充、生活感 |
| 环境音过渡 | 2-5 秒 | 场景切换、情绪重置 |
| 功能揭晓瞬间 | 2-3 秒 | 让观众"哇"出来 |
| 角色表情反应 | 2-4 秒 | 情感共鸣 |
| 音乐独奏 | 5-15 秒 | 情绪渲染、氛围建立 |

### 旁白密度控制原则

**高密度旁白**（信息传递）：
- 版本号、发布时间
- 功能名称和一句话解释
- 操作步骤说明

**低密度旁白**（情感体验）：
- 角色互动场景
- 风景/建筑展示
- 音乐主导段落

**黄金比例**：
- 免费内容堆叠段：旁白占 60-70%，留白 30-40%
- 付费内容深度段：旁白占 40-50%，留白 50-60%

## 台词风格规范

### 口吻定位

**不是**：营销人员、新闻主播、攻略博主
**是**：带你参观的老朋友、岛上的邻居、热情的导游

### 句式特征

**短句优先**：
```
❌ "The Roost is a café that is owned by Brewster, who is a pigeon character..."
✅ "This is the Roost, a café, owned by the composed Brewster."
```

**逗号切割**：
```
"Kick back and relax, while Brewster serves you up a cup of his finest, hand-selected coffee."
```

**形容词前置**：
```
"the composed Brewster"
"his pride and joy"
"mysterious flora"
```

### 关键词库

**邀请/建议**：
- Why not...?
- Consider...
- Try out...
- Why not give...?

**情感反应**：
- Mmm, looks tasty!
- It looks fantastic!
- Seems like the client is pretty happy!

**过渡/连接**：
- Speaking of...
- In addition...
- Plus, ...
- And let's not forget...

**结尾升华**：
- savor
- relish
- enjoy
- experience

## Nintendo Direct 台词风格执行指南（中文本土化版）

### 核心原则

Nintendo Direct 的台词风格不是"翻译腔"，而是一种**"导游带你参观"的口吻**——热情但不亢奋，亲切但不随意，专业但不高冷。中文版本需要保持这种"有节制的热情"。

### 句式模板库（可直接套用）

#### 介绍新角色/新NPC
```
模板：「这是[名字]。[一句话定位/在哪里找到Ta]。」

示例：
- 「这是鸽巢。你会在小镇的码头边找到他，懒洋洋地晒着太阳。」
- 「这是朵朵，小镇里最懂时尚的猫咪。」
```

#### 介绍新地点/新场景
```
模板：「这是[名称]，一个[类型]，[定语描述]。」

示例：
- 「这是咕咕咖啡馆，一间温馨的小店，由沉稳的鸽巢经营。」
- 「这是神秘沙滩，一片只有老水手才知道的秘境。」
```

#### 介绍新玩法/新功能
```
模板：「[功能]来了！」或「[功能]，即将加入[游戏名]。」

示例：
- 「烹饪玩法来了！」
- 「家园设计，即将加入你的小镇生活。」
- 「躲猫猫玩法，将在下周上线。」
```

#### 邀请体验
```
模板：「不妨[动作]？」「要不要[动作]？」「来[动作]吧。」

示例：
- 「不妨坐下来，慢慢品尝一杯手冲咖啡？」
- 「要不要和朋友一起，享受这段悠闲时光？」
- 「来布置属于你的小角落吧。」
```

#### 情感点评（短促、即时）
```
模板：「嗯，[感官描述]！」「[形容词]！」

示例：
- 「嗯，看起来真香！」
- 「太棒了！」
- 「完美。」
```

**注意：** 即使是短促点评，也是完整句子（主语可省略，但谓语完整），不是孤立形容词。

#### 延伸想象
```
模板：「说不定[可能性]...」「也许有一天[远景]...」「谁知道呢[悬念]...」

示例：
- 「说不定，你会在这里遇到意想不到的朋友...」
- 「也许有一天，你也能成为小镇的设计大师...」
- 「谁知道呢，下一个惊喜就在转角等着你。」
```

### 中文表达技巧

#### 1. 短句优先，逗号切割

短句 = 主谓宾完整、意思明确的完整句子，不是碎片化关键词。

```
❌ 复杂句：「咕咕咖啡馆是由鸽巢经营的一间温馨小店，你可以在这里品尝到各种手冲咖啡。」
✅ Direct风格：「这是咕咕咖啡馆，一间温馨的小店，由沉稳的鸽巢经营。来品尝一杯手冲咖啡吧。」
```

**关键区分：**
- ✅ 短句："变身之后，你会融入周围的环境。"（主谓宾完整）
- ❌ 碎片化："变身。融入环境。"（只有关键词，无语法结构）

#### 2. 形容词前置

```
❌ 后置：「鸽巢是一个沉稳的角色...」
✅ 前置：「沉稳的鸽巢...」「神秘的沙滩...」「温馨的小店...」
```

#### 3. 少用"你"，多用"你的"

```
❌ 「你可以在这里做...」
✅ 「在你的小镇里...」「你的家园...」「你的朋友...」
```

#### 4. 邀请而非指令

```
❌ 指令：「快来试试吧！」「马上开始！」
✅ 邀请：「不妨试试看？」「来体验一下吧。」「要不要来一场？」
```

## 心动小镇适配备注

### 可直接沿用的元素

- 三明治五段式结构
- 波浪式情绪节奏
- 旁白-画面留白比例
- 导游式口吻和句式

### 需要替换的元素

| 动森元素 | 心动小镇替换 |
|----------|--------------|
| Animalese（动物语） | 小镇居民的说话方式（如有）或简化为表情动作展示 |
| Brewster/Kapp'n 等经典角色 | 小镇自有 NPC（如朵朵、阿塔拉等） |
| 博物馆/咖啡厅等地点 | 小镇自有场景（如海边、广场、商店街） |
| 岛屿设定 | 小镇/家园设定 |
| Nook Miles | 小镇自有货币/积分系统 |

### 音乐风格适配

- 保持"原声吉他/钢琴"为主的治愈系基调
- 可根据小镇具体场景调整（如海边用更多海浪声、森林用鸟鸣声）

## 时长分配参考

### 完整版（20-25 分钟）

| 段落 | 时长 | 内容量 |
|------|------|--------|
| 开场钩子 | 30-45 秒 | 问候 + 主题 + 发布时间 |
| 免费内容堆叠 | 10-12 分钟 | 8-12 个功能点，每个 45-90 秒 |
| 情感转折 | 15-30 秒 | 最后免费更新声明 + 悬念钩子 |
| 付费内容深度 | 8-10 分钟 | 1-2 个核心玩法，完整流程展示 |
| 收尾号召 | 30-45 秒 | 感谢 + 情感升华 + 发售日 |

### 精简版（5-8 分钟）

| 段落 | 时长 | 内容量 |
|------|------|--------|
| 开场钩子 | 15-25 秒 | 精简问候，快速进入主题 |
