# 结构：快剪版 (fast_cuts)

适合 1–2 分钟的短视频、快节奏剪辑，每条更新一句话带过。

## 段落顺序与要求

| 顺序 | 段落 id | 说明 | 建议时长/篇幅 |
|------|---------|------|----------------|
| 1 | hook | 3 秒钩子：一句吸引注意（如「X 版本来了，X 条更新速看」） | 约 3–5 秒 / 15–25 字 |
| 2 | bullets | 要点罗列：每条更新一行或一句，不展开；可用「• 新英雄 XXX」「• 某英雄削弱」等列表式 | 每条 5–15 字，总条数视内容 5–15 条 |
| 3 | cta | 收尾：一句 CTA（如「完整 patch 见简介」「点赞不迷路」） | 约 3–5 秒 / 10–20 字 |

## 写作要点

- **极简**：不解释原因、不报具体数值（除非用户要求），只传达「改了什么」。
- **可剪成字幕**：每一条适合单独成字幕卡或画面标题，便于快剪。
- 若 changelog 内容很多，只保留「最吸睛」的 8–12 条（新角色、重大平衡、重要 Bug），其余用「还有若干小改动」一句概括。

## 时间戳（可选）

快剪版通常按条切镜，可在每条前加 `[MM:SS]` 占位，每条约 3–8 秒，例如：

- `[0:00]` hook  
- `[0:05]` 第 1 条  
- `[0:10]` 第 2 条  
- …

## 与风格的配合

- 风格仍适用：formal 则列表也用规范用语；casual/funny 则可用梗或口语（如「某英雄挨刀」）。
- 整篇总字数建议控制在 200–400 字（中文），便于 1–2 分钟口播。
