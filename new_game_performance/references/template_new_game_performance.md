# 新游表现报告模板

本模板用于 `new_game_performance` skill。按以下 section 顺序生成报告，每个 section 均需填写；若某项数据无法获取，注明「数据不足」而非跳过该 section。引用格式遵循 `citation_instructions.md`（与 game_report 一致），凡引用具体数据或引文必须附 [N]，报告末尾输出「引用来源」章节。

---

## 报告头部（不作为 section 编号）

```
# {游戏名称} 新游表现报告

**报告日期：** {YYYY年M月D日}
**数据来源：** {实际使用的平台列表，如「TapTap、Reddit」}
**分析时段：** 上线后 {N} 天内（{上线参考日} 起）
**分析师：** 新游表现研究助手

---
```

---

## 一、执行摘要

用表格呈现上线期核心指标，让读者在 5 秒内了解该新游的首周表现。

**格式：**

```
| 指标 | 数值 | 说明 / 预警 |
|------|------|-------------|
| **上线参考日** | {YYYY年M月D日} | {依据：应用商店/论坛首波讨论/评价时间分布} |
| **首周综合评分** | {X.X}/10 | {与首日对比：↑/↓/→ ±X.X} |
| **首周评价总数** | {X,XXX} 条 | - |
| **下载量 / 关注数** | {若有：XXX 万；若无：暂无} | - |
| **整体结论** | {1～2 句话：如「首周口碑中等偏上，早期差评集中在崩溃与适配，建议优先热修」} | 🟢/🟡/🔴 |
```

**填写指引：**
- 整体结论预警：🟢 稳定、🟡 需关注、🔴 需尽快干预（参考 launch_knowledge 的归因与时限）。
- 若无法确定上线日，在「上线参考日」行注明推断依据。

---

## 二、上线节点与首周指标

呈现可获得的量化指标随时间的变化（首日 / 第 3 天 / 第 7 天等，按实际数据粒度填写）。

**格式：**

```
| 时间点 | 评分 | 评价数 | 下载/关注（若有） | 备注 |
|--------|:----:|:------:|:-----------------:|------|
| 首日   | {X.X} | {XXX} | {若有}            | {如：开服高峰} |
| 第 3 天 | {X.X} | {XXX} | {若有}            | - |
| 第 7 天 | {X.X} | {XXX} | {若有}            | - |
```

**填写指引：**
- 若仅有「当前」一个时间点，则一行即可，备注中说明「仅单点数据」。
- 趋势判断：评分持续下行需在「风险与建议」中展开；波动大但样本少时注明「样本有限，结论谨慎参考」。

---

## 三、早期口碑与差评集中点

归纳上线期玩家好评与差评的主要维度，差评需按 launch_knowledge 的归因层（技术/商业化/运营/策划）归类，并附典型引用 [N]。

**格式：**

```
### 好评集中点
- **{维度1}**：{1 句话概括} — 代表引用：「{引文}」[N]
- **{维度2}**：…

### 差评集中点（按归因层）
| 归因层 | 问题摘要 | 典型引用 | 严重程度 |
|--------|----------|----------|:--------:|
| 技术层 | {如：崩溃/闪退/机型适配} | 「{引文}」[N] | 高/中/低 |
| 商业化层 | {若有} | [N] | - |
| 运营层 | {若有} | [N] | - |
| 策划层 | {若有} | [N] | - |
```

**填写指引：**
- 每个 blockquote 或典型引用必须附 [N]，与报告末尾「引用来源」一一对应。
- 严重程度结合提及频次与对留存/评分的影响判断（见 launch_knowledge）。

---

## 四、与同期新游或竞品对比（可选）

若数据中有同品类同期新游或明确竞品的 app_info（评分、下载/关注），可做简要对比表；若无则本 section 写「本期暂无对比数据」，不臆测。

**格式：**

```
| 游戏 | 上线时间 | 首周评分 | 首周评价数 | 备注 |
|------|----------|:--------:|:----------:|------|
| {本游戏} | {日期} | {X.X} | {XXX} | - |
| {竞品/同期新游} | {日期} | {X.X} | {XXX} | {数据来源} |
```

---

## 五、风险与建议

按 launch_knowledge 的「风险与建议输出要求」：按归因层分类，每条含问题、依据、建议、时限（立即/本版本内/下版本前）。

**格式：**

```
### 技术层
- **[问题]**：{简述}  
  **依据**：{数据或引用 [N]}  
  **建议**：{如热修、兼容性排查}  
  **时限**：立即 / 本版本内 / 下版本前

### 商业化层
（若有则同上格式）

### 运营层
（若有则同上格式）

### 策划层
（若有则同上格式）
```

---

## 六、引用来源

在报告最后必须输出本章节，格式与 game_report 的 `citation_instructions.md` 一致：仅列出正文中**实际引用过**的编号，每条一行，包含编号、类型、标题/摘要、平台、日期、URL。

示例：
```
## 引用来源
- [1] review | {正文前 30 字} | TapTap | {时间} | {url}
- [2] forum_post | {标题前 30 字} | Reddit | {时间} | {url}
```
