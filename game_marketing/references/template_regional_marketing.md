# 地区市场营销方案模板

本模板用于 `game_marketing` skill 在 **task_focus = marketing_plan** 时的输出。按以下 section 顺序生成；对每个地区重复「二、分地区方案」中的区块。若某项无数据或需业务方补充，注明「待定」或「需补充」。

---

## 报告头部

```
# {游戏名称} 地区市场营销方案

**报告日期：** {YYYY年M月D日}
**本品：** {游戏名}
**覆盖地区：** {地区列表，如 中国大陆、日本、东南亚}

---
```

---

## 一、竞品与 STP 摘要（全局）

- **竞品列表**：来自 market_landscape 的竞品名单及一句话简介。
- **核心用户画像（STP 结论）**：核心用户占比、典型描述、兴趣标签、触媒习惯概要。
- **游戏卖点**：3–5 条，应对市场竞争的差异化卖点。

---

## 二、分地区方案

以下区块对 **regions** 中每个地区各写一节。可根据当地市场微调用户画像与渠道；无数据处注明「待定」或「需补充」。

### 【地区】：{如 中国大陆 / 日本 / 东南亚}

**【目标】**
- 首月下载量：…
- 收入目标：…
- 品类排名目标：…

**【用户画像】**
- 核心用户（占比 X%）：[具体描述]
- 兴趣标签：[tag1, tag2, tag3]
- 触媒习惯：[平台偏好]

**【渠道策略】**
- 主力渠道：[如 TikTok/YouTube/本地渠道]
- 预算占比：X%
- 内容形式：[短视频/直播/图文 等]

**【本地化重点】**
- 文化适配：[具体调整点]
- 支付方式：[本地钱包接入]
- KOL 合作：[层级与数量]

**【里程碑】**
- D-30：预注册开启，目标 X 万
- D-Day：冲榜活动，买量峰值
- D+30：首版本活动，留存优化

**【联动规划】**（填写时参考 references/collaboration_framework.md；须含用户群体匹配考量）
- 建议联动类型：[IP/品牌/音乐/文旅/节日/KOL 等，按地区择要]
- 与目标用户画像的匹配理由：[1 句，说明为何该联动方向与本品该地区用户画像匹配]
- 优先方向与档期：[如 D+30 首版本 IP 联动；与用户画像、渠道策略一致]
- 风险或前提：[如 需提前确认授权与档期；联动方舆情预检]

---

## 三、数据与局限

- **数据范围**：本品与竞品使用的平台、数据类型、时间范围。
- **局限说明**：方案中的目标与预算等多为基于竞品与 STP 的推断或占位，供战略/市场参考；具体数值需业务方确认。

---

## 文件名建议

输出为文件时，建议命名：`{YYYY-MM-DD}_{游戏名称}_地区营销方案.md`。
