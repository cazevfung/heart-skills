# 游戏群像KV构图 Skill

本 Skill 提供基于商业目的的游戏群像KV构图体系，帮助生成符合行业标准的游戏宣传图构图指引。

## 重要警告（Cazev 的明确要求）

### 给豆包写 Prompt 时绝对不要做：
1. **不要写抽象指令** — 不要出现"零站桩"、"明显动作"、"有互动"这类词，豆包不会"想"
2. **不要写复杂层级规则** — 不要把"前景占60%、中景占30%"这类技术说明写进 prompt
3. **不要丢给豆包构思** — 必须先自己想清楚画面，再写 prompt

### 正确的 Prompt 写法：
1. **先自己构思清楚**：有几个角色、在什么位置、什么 pose、怎么互动
2. **简洁描述最终画面**：只写画面里有什么，不写怎么实现
3. **具体到肢体动作**：写"右腿弯曲90度"、"左手牵着对方的手"、"身体向左倾斜"
4. **具体到互动细节**：写"手牵手一起跳"、"肩膀靠在一起"、"视线交汇"

## 四类构图体系（用于我自己构思，不写进 prompt）

### 一、公测群像类（Launch Key Visual）

**核心目的**：建立游戏世界观初印象，展示完整角色资产，营造"入局"氛围感。

**我自己构思时的参考**：
- 正三角或对称式构图
- 主角占25-30%，其余角色均分
- 背景40%以上，仰视角度
- 3+1疏密法则

**Prompt 示例**（简洁版）：
```
主角位于画面右侧，从头到脚占画面高度70%，双腿离地跳跃，左手牵着左侧角色，右手张开。左侧角色占画面高度50%，单腿跳跃，右手被主角牵着，左手挥舞彩带。背景是清澈蓝天和白云。
```

---

### 二、地图展示类（Environment/Map Focus）

**核心目的**：新区域开放或世界观扩展，角色服务于场景。

**我自己构思时的参考**：
- 框架式+三分法
- 场景占60-70%，角色小于30%
- 强制透视，纵深引导

---

### 三、新角色登场类（New Character Reveal）

**核心目的**：在已有角色矩阵中突出新个体。

**我自己构思时的参考**：
- 中心聚焦式+向心式
- 新角色占40-50%，位于黄金分割中心
- 旧角色环绕，视线汇聚
- 虚实对比

---

### 四、新皮肤/时装类（Skin/Costume Showcase）

**核心目的**：展示服装细节与特效质感。

**我自己构思时的参考**：
- 垂直线构图或对角线张力
- 单/双角色占60-70%
- 背景极简
- 动态pose展示服装

---

## 视觉权重对比表（用于我自己构思）

| 类别 | 角色总占比 | 背景占比 | 主次对比度 | 核心几何构图 |
|------|------------|----------|------------|--------------|
| 公测群像 | 60% | 40% | 中（主角:配角约1.5:1） | 正三角对称 |
| 地图展示 | 30% | 70% | 低 | 框架式、三分法 |
| 新角色登场 | 70% | 30% | 极高（新:旧约3:1） | 中心聚焦 |
| 新皮肤类 | 75% | 25% | 极高 | 垂直线、对角线 |
