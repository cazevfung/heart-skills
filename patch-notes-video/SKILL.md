# Patch Notes Video Skill

本skill不提供固定模板，而是提供**设计原则**与**决策框架**，帮助创作者根据具体情境构建独特的前瞻视频结构。

---

## 核心设计原则

### 原则1：情绪先于信息

**核心问题：** 观众在看完视频后应该**感受到什么**，而非仅仅**知道了什么**。

**思考维度：**
- 这个版本的情绪基调是什么？（兴奋/感动/悬念/怀旧）
- 观众当前对游戏的情绪状态？（期待/疲惫/焦虑/满足）
- 我们希望视频结束后观众处于什么情绪？

**反模式警示：**
❌ 把前瞻变成"功能清单朗读"——信息正确但情绪 flat
❌ 高潮点放在最后但前面已经信息过载——观众疲惫无法接收

**正确做法：**
- 每个信息点都服务于情绪目标
- 高潮点之前要有"情绪爬坡"
- 结尾情绪与开场形成呼应或升华

---

### 原则2：节奏即内容

**核心问题：** 信息的**释放节奏**本身就是叙事的一部分。

**思考维度：**
- 哪些信息需要"慢慢品味"？（角色故事、世界观展开）
- 哪些信息需要"快速轰炸"？（福利、数值、机制）
- 哪里需要"停顿留白"？（悬念、情感高潮后）

**决策框架：**

| 节奏类型 | 适用场景 | 手法 | 风险 |
|----------|----------|------|------|
| 慢速沉浸 | 剧情向、情怀向 | 长镜头、音乐铺垫、情感台词 | 容易拖沓 |
| 中速推进 | 标准内容介绍 | 对话节奏、适度演示 | 容易平庸 |
| 快速切割 | 福利、机制、数据 | 快剪、字幕强调、数据可视化 | 容易信息过载 |
| 停顿留白 | 悬念、情感转折 | 黑屏、音乐骤停、特写 | 容易冷场 |

**关键判断：** 节奏变化本身创造情绪起伏。一段全程匀速的视频，即使内容再好也会让人疲惫。

---

### 原则3：观众分层设计

**核心问题：** 同一视频如何同时服务**不同深度**的观众？

**玩家分层：**

| 层级 | 特征 | 需求 | 设计策略 |
|------|------|------|----------|
| 云玩家/流失玩家 | 看热闹、追热点 | 情绪价值、社交货币 | 前3分钟必须有爆点 |
| 轻度活跃玩家 | 偶尔上线、关注福利 | 明确收益、低门槛参与 | 福利信息前置、清晰 |
| 核心玩家 | 每日活跃、关注强度 | 机制深度、配队策略 | 中段硬核内容 |
| 硬核/攻略向 | 数据敏感、追求最优 | 数值细节、极限场景 | 可跳过的深度解析 |

**设计策略：**
- **分层释放**：同一内容用不同深度呈现（如角色：PV→故事→机制→数值）
- **可跳过设计**：硬核内容明确标识，轻度玩家可跳过不损失主线体验
- **多入口设计**：不同层级观众都能在前3分钟找到吸引自己的点

**反模式警示：**
❌ 全程硬核——轻度玩家流失
❌ 全程浅层——核心玩家觉得"没东西"
❌ 试图讨好所有人——结果所有人都不满意

---

### 原则4：传播预设

**核心问题：** 视频发布后，**什么会被传播**？

**传播单元设计：**

| 类型 | 特征 | 设计要点 |
|------|------|----------|
| 金句 | 易引用、有态度 | 口语化、有节奏感、带情绪 |
| 名场面 | 视觉冲击、情绪高点 | 3-5秒可独立成片的画面 |
| 福利爆点 | 数字明确、对比强烈 | "免费""X抽""限定"等关键词 |
| 悬念钩子 | 信息缺口、引发讨论 | 点到为止，不解释清楚 |
| 情怀callback | 老玩家共鸣、集体记忆 | 经典元素重现、时间跨度 |

**关键判断：** 不是"希望什么被传播"，而是"什么**必然**会被传播"。传播点是设计出来的，不是碰运气。

## 结构决策框架

### 决策树：如何选择开场方式

```
版本是否有强情绪主题？
├── 是（如：周年庆、大结局、角色告别）
│   └── 选择：情感钩子
│       - 情怀callback开场
│       - 角色/玩家故事引入
│       - 悬念情绪铺垫
│   
└── 否（常规版本更新）
    └── 版本是否有核心爆点？
        ├── 是（新角色/新玩法极具吸引力）
        │   └── 选择：爆点前置
        │       - 直接展示最吸睛内容
        │       - 3秒内抓住注意力
        │
        └── 否（内容平均、缺乏单一爆点）
            └── 选择：氛围建立
                - 世界观/主题氛围铺垫
                - 主持人/角色建立连接
```

---

### 决策树：高潮点位置设计

```
版本核心卖点数量？
├── 单一强卖点（如：只有一个新角色但极其重要）
│   └── 高潮点：中后段（2/3处）
│       - 前段铺垫情绪
│       - 中段逐步释放信息
│       - 高潮完整呈现+情感升华
│
└── 多卖点并列（如新角色+新地图+新系统）
    └── 卖点之间是否有递进关系？
        ├── 是（有叙事/情绪递进）
        │   └── 高潮点：阶梯式设计
        │       - 小高潮1 → 小高潮2 → 大高潮
        │       - 每个高潮后短暂回落再爬升
        │
        └── 否（卖点独立、无递进）
            └── 高潮点：前置爆发
                - 最强卖点前置（前10分钟）
                - 中段填充其他内容
                - 结尾福利收尾
```

---

### 决策树：风格定位

```
游戏核心体验是？
├── 角色/情感驱动（如：原神、崩坏系列）
│   └── 风格：陪伴感/茶会感
│       - 真人/角色建立"朋友"关系
│       - 闲聊、互动、情感分享
│       - 信息包裹在情感交流中
│
├── 策略/机制驱动（如：星铁、明日方舟）
│   └── 风格：专业感/发布会感
│       - 主持人作为"讲解员"
│       - 机制拆解、数值分析
│       - 信息密度优先
│
├── 社交/派对驱动（如：蛋仔、元梦）
│   └── 风格：热闹感/派对感
│       - 多人互动、游戏化环节
│       - 高密度梗、快节奏
│       - 参与感、即时反馈
│
└── 叙事/IP驱动（如：1999、方舟剧情向）
    └── 风格：沉浸感/剧场感
        - 叙事主导、电影化语言
        - 世界观展开、悬念铺设
        - 观众作为"故事参与者"
```

## 反模式库（常见陷阱）

### 陷阱1：模板依赖症

**症状：** 每个版本都用同样的结构、同样的节奏、同样的转场。

**后果：** 观众产生"审美疲劳"，失去期待感。

**解药：**
- 每个版本问自己：这个版本的**独特情绪**是什么？
- 结构服务于情绪，而非反过来
- 允许"打破常规"的版本存在

---

### 陷阱2：信息倾倒

**症状：** 试图在有限时间内塞入所有更新内容。

**后果：** 观众什么都听了，什么都没记住。

**解药：**
- 区分"必须知道"和"可以不知道"
- 次要信息放图文公告，视频聚焦核心
- 用"少即是多"原则：减少信息点，加深每个点的印象

---

### 陷阱3：自说自话

**症状：** 只从开发者视角讲"我们做了什么"，不考虑"玩家关心什么"。

**后果：** 玩家感觉"与我无关"。

**解药：**
- 每个信息点都问：玩家**为什么**要关心这个？
- 用玩家语言，而非开发语言
- 建立"玩家收益"与"功能特性"的明确连接

---

### 陷阱4：高潮错位

**症状：** 最强内容放在最后，但观众前面已经流失。

**后果：** 核心卖点未被有效传播。

**解药：**
- 前3分钟必须有"钩子"
- 核心卖点在观众注意力峰值时呈现（通常15-25分钟）
- 结尾是"情感收束"而非"信息释放"

## 动态适配方法

### 情境适配：版本类型

| 版本类型 | 核心挑战 | 结构策略 | 情绪重点 |
|----------|----------|----------|----------|
| **大版本/周年庆** | 内容太多、情绪复杂 | 情怀回顾+新内容+未来展望三段式 | 感恩+期待 |
| **常规版本** | 缺乏爆点、容易平淡 | 单一主线贯穿、其他内容辅助 | 新鲜感 |
| **过渡版本** | 玩家期待低 | 爆点前置、缩短时长、强调福利 | 惊喜感 |
| **危机回应** | 玩家情绪负面 | 直面问题、解决方案、诚意展示 | 信任重建 |

---

### 情境适配：游戏生命周期

| 阶段 | 玩家状态 | 前瞻重点 | 避免 |
|------|----------|----------|------|
| **上线期** | 好奇、试探 | 世界观建立、核心玩法展示 | 过度承诺 |
| **成长期** | 热情、投入 | 内容深度、社区互动 | 忽视反馈 |
| **成熟期** | 习惯、挑剔 | 创新点、情怀维护 | 重复套路 |
| **衰退期** | 疲惫、流失 | 重大变革、回归福利 | 粉饰太平 |

## 执行工作流

不提供固定步骤，而是提供**思考清单**：

### 阶段1：理解情境
- [ ] 这个版本在游戏生命周期中的位置？
- [ ] 玩家当前对游戏的整体情绪状态？
- [ ] 版本本身的情绪基调是什么？
- [ ] 核心卖点是什么？（只能选一个）

### 阶段2：设计情绪弧线
- [ ] 开场情绪目标？（好奇/感动/兴奋/悬念）
- [ ] 高潮点位置？（前/中/后/阶梯）
- [ ] 结尾情绪目标？（期待/满足/留恋/行动）
- [ ] 情绪转折点如何设计？

### 阶段3：选择结构策略
- [ ] 开场方式？（情感钩子/爆点前置/氛围建立）
- [ ] 节奏组合？（慢-快-慢 / 快-慢-快 / 阶梯）
- [ ] 观众分层如何覆盖？
- [ ] 传播点预设？

### 阶段4：填充内容
- [ ] 每个信息点服务于什么情绪目标？
- [ ] 哪些可以删减？
- [ ] 哪些需要深化？
- [ ] 转场如何保持情绪连贯？

### 阶段5：验证与调整
- [ ] 前3分钟是否有足够吸引力？
- [ ] 高潮点是否清晰可感知？
- [ ] 结尾是否有记忆点？
- [ ] 整体是否有"意外"元素？

## 关键提醒

**本skill的核心价值不在于提供答案，而在于提供思考问题的方式。**

每个版本都是独特的，每个游戏都有自己的语境。模板是起点，理解原则才能超越模板。

最终检验标准：**看完视频后，观众是否感受到了我们希望他们感受到的？**
