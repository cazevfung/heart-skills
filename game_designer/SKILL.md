# 游戏设计师 Skill

本 skill 是**品类设计知识层**。不直接爬取或产出报告，而是提供国内主流游戏品类的设计要素与设计预期基线，供 agent 在分析、报告、文案创作时注入品类视角。品类为**维度**，可重叠（如开放世界×RPG×二次元）；每个维度有独立文档，按 [references/_index.md](references/_index.md) 定位后**仅加载相关品类文件**，专业化且省 token。

## 何时使用

**独立使用：**
- 用户问：某品类怎么设计、核心循环是什么、该品类常见设计陷阱、新游立项/玩法设计建议
- 用户提到：游戏设计、玩法设计、核心循环、设计要素、品类设计、Gacha 设计、开放世界设计

**与其他 skill 联合使用：**
- 用户在做付费/留存/舆情/公告分析时，希望报告或文案带有**品类设计视角**（该品类玩家期待什么、设计基线是什么、哪些设计风险需关注）
- 其他 skill 的上下文中出现品类关键词（如二次元、开放世界、卡牌、SLG），agent 可主动读取本 skill 以丰富分析维度

## 品类推断

在未明确指定品类时，按以下优先级推断（可多选，因品类可重叠）：

1. **用户消息**：用户直接说出「开放世界」「卡牌」「二次元」「SLG」等 → 取用户用词对应维度
2. **游戏名 / 已知产品**：若任务涉及具体游戏，从 **read_game_registry** 返回的注册表或 **read_game_data** 读取的 `app_info` 中获取应用描述、标签、分类字段，对照 references/_index.md 中的触发关键词映射到品类及对应文件名
3. **默认**：无法推断时，在输出中注明「未指定品类，采用通用设计视角」；若后续分析需要品类基线，可询问用户或列出可选品类供用户选择

**映射示例：** 应用描述/标签中出现「抽卡」「角色收集」「二次元」→ 至少包含 Gacha/卡牌、二次元；「大世界」「开放世界」「探索」→ 开放世界；「回合制」「养成」→ RPG/养成等。

## 与其他 skill 的协同

| 场景 | 使用 skill | 本 skill 的贡献 |
|------|------------|-----------------|
| 付费/变现深度分析 | **game_monetization** | 提供品类维度的付费分析重点、典型风险与健康度预期（见对应品类文档「与商业化/留存的交叉点」），报告可增加「品类设计视角下的付费健康度」 |
| 留存/流失分析 | **game_retention** | 提供该品类的核心留存循环与常见流失设计陷阱，报告可增加「品类设计视角下的留存风险」 |
| 舆情/版本/综合报告 | **game_report** | 提供品类设计预期基线，将玩家反馈与「该品类应有的设计要素」对照，归纳是否偏离品类预期 |
| 公告文案撰写 | **game_announcement_copy** | 提供该品类玩家核心动机与敏感点，文案突出品类关心的卖点、规避品类常见雷区 |
| 公告配图/视觉 | **game_announcement_image** | 提供品类调性（如二次元、写实、休闲），指导风格与信息层级 |

联合使用时：先按上述场景执行主 skill 的流程，在需要「品类视角」的环节**先读 references/_index.md** 定位品类文件名，**仅读匹配到的品类文档**（可多个），再注入到输出中；不读与当前任务无关的品类文件。

## 输入/输出丰富规则

当本 skill 与任一分析/创作 skill 同时被读取且任务涉及具体游戏或品类时：

1. **输入丰富**：在分析前，先读 references/_index.md 确定要加载的品类文件，**仅读**这些文件，从中取出「核心玩法循环」「关键设计系统」「玩家核心动机」「常见设计陷阱」「与商业化/留存的交叉点」，作为分析时的设计基线上下文。
2. **输出丰富**：在报告或文案中增加 **「品类设计视角」** 小节（或等价段落），内容可包括：
   - 本任务涉及的品类维度及简要定义
   - 该品类下玩家对设计/商业化/留存的普遍预期
   - 当前分析结论与品类设计基线的对照（是否吻合、是否存在品类典型风险）
   - 若为公告类产出：该品类玩家最关心的卖点与需规避的表述

若用户未指定品类且无法从数据推断，则输出中注明「未指定品类，未注入品类设计视角」，不编造品类。

## 参考文件与加载规则

1. **必读** [references/_index.md](references/_index.md)：品类关键词与文件名的映射表，用于确定当前任务需加载的品类文件列表。
2. **按需读** 匹配到的品类文件：仅读 _index 中与当前推断品类对应的 `genre_*.md`（可多个，因品类可重叠）。
3. **若需品类组合视角**：加读 [references/genre_combinations.md](references/genre_combinations.md)。
4. **不读**与当前任务无关的品类文件，以专业化且减少 token 消耗。

## 使用注意

- 本 skill 仅提供知识与规则，不替代 game_crawl、game_report 等 skill 的数据获取与报告生成职责。
- 品类名称与 references 内各品类文档的术语保持一致（如统一用「Gacha/卡牌」「开放世界」「二次元」），便于与其他 skill 的文档交叉引用。
