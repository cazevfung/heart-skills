# 品类组合参考

当任务涉及**多个品类维度**（如开放世界×RPG、二次元×Gacha）时，先分别读取各维度对应的 genre_*.md，再结合本表检查组合特有的设计张力与风险。

| 组合示例 | 叠加注意点 |
|----------|------------|
| **开放世界 × RPG** | 探索节奏与剧情推进的平衡；大世界内容消耗与产能；成长曲线与世界难度匹配。 |
| **二次元 × Gacha × RPG** | 角色情感与强度平衡；剧情与抽卡节奏；社区与二创对留存的放大效应。 |
| **塔防 × Gacha** | 关卡策略与角色强度的平衡；抽卡不破坏关卡解法多样性；活动与限时关卡节奏。 |
| **放置 × 卡牌/Gacha** | 挂机收益与抽卡资源的平衡；免费与付费进度差距；长线阵容深度与策略。 |
| **MOBA × 休闲** | 单局时长与深度取舍；公平性与轻度付费的兼容；社交与单局体验。 |
| **开放世界 × 生存/建造** | 探索与生存压力、建造自由度的三角平衡；性能与内容密度。 |
| **Roguelite × 卡牌构筑** | 单局随机与构筑深度的结合；永久成长不削弱单局决策价值。 |

分析多品类组合时：先分别套用各维度文档中的「核心玩法循环」「关键设计系统」「常见设计陷阱」与「与商业化/留存的交叉点」，再根据上表检查组合特有的张力与风险，并在报告中注明所采用的品类维度及组合。
