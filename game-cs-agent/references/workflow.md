## 状态流转详解

### 状态定义

```
STATE_IDLE          - 空闲状态，等待玩家输入
STATE_GUIDE_QA      - 攻略问答状态
STATE_BUG_INTAKE    - Bug 收集状态
STATE_CLARIFYING    - 澄清追问状态
STATE_PENDING       - 等待玩家回复
```

### 状态转换表

| 当前状态 | 玩家输入 | 意图识别 | 下一状态 | 动作 |
|---------|---------|---------|---------|------|
| IDLE | "小麦怎么获得" | guide_qa (0.9) | GUIDE_QA | 调用 guide_qa |
| IDLE | "商店不见了" | bug_intake (0.9) | BUG_INTAKE | 调用 bug_intake |
| IDLE | "不生长" | unclear (0.4) | CLARIFYING | 生成澄清问题 |
| GUIDE_QA | "玉米呢" | guide_qa (0.8) | GUIDE_QA | 保持状态，回答 |
| GUIDE_QA | "商店不见了" | bug_intake (0.9) | BUG_INTAKE | 切换状态 |
| BUG_INTAKE | "小麦怎么获得" | guide_qa (0.9) | GUIDE_QA | 保存进度，切换 |
| BUG_INTAKE | "iPhone14" | continue_bug | BUG_INTAKE | 继续收集 |
| CLARIFYING | "浇水了还是不行" | bug_intake (0.8) | BUG_INTAKE | 确认后切换 |

## 消息格式规范

### CS Agent → guide_qa

```json
{
  "action": "query",
  "query": "小麦怎么获得",
  "context": {
    "last_topic": "crops",
    "session_id": "xxx"
  }
}
```

guide_qa → CS Agent
```json
{
  "status": "success",
  "answer": "小麦可以通过...",
  "confidence": 0.95,
  "related_topics": ["玉米", "种植"]
}
```

### CS Agent → bug_intake

```json
{
  "action": "start_intake",
  "initial_description": "商店图标不见了",
  "context": {
    "session_id": "xxx"
  }
}
```

bug_intake → CS Agent (第一轮)
```json
{
  "status": "in_progress",
  "stage": 1,
  "next_question": "是在哪个界面看不到商店？",
  "collected": {
    "title": "商店图标消失",
    "type": "visual_bug"
  }
}
```

CS Agent → bug_intake (继续)
```json
{
  "action": "continue_intake",
  "player_response": "主界面",
  "stage": 1
}
```

bug_intake → CS Agent (完成)
```json
{
  "status": "complete",
  "bug_data": {
    "uid": "123",
    "title": "商店图标消失",
    "type": "visual_bug",
    ...
  },
  "storage_result": "success"
}
```

## 边界情况处理

### 情况 1：玩家同时说两件事

```
玩家: "小麦怎么获得，还有我的商店不见了"

处理:
1. 检测到两个意图
2. 优先级: Bug > 攻略
3. 先处理 Bug: "先帮你看看商店的问题..."
4. Bug 完成后: "刚才还问了小麦，小麦可以通过..."
```

### 情况 2：玩家在 Bug 收集中说"算了"

```
玩家: "算了，不报了"

处理:
1. 检测到取消意图
2. 保存已收集信息（可选）
3. 确认: "好的，已取消。如果问题还在随时找我"
4. 状态回到 IDLE
```

### 情况 3：玩家重复提问

```
玩家: "小麦怎么获得"
玩家: "小麦怎么获得" (重复)

处理:
1. 检测重复
2. 简化回答: "小麦可以通过种植、商店购买、好友赠送获得。需要详细说明吗？"
```

### 情况 4：玩家输入无法识别

```
玩家: "asdfghjkl"

处理:
1. 所有意图置信度 < 0.3
2. 回复: "抱歉，我没太明白。你可以问：
   - 攻略问题：XX怎么获得
   - Bug反馈：我遇到了XX问题"
```

## 性能优化

### 缓存策略

```
1. 知识库缓存: 启动时加载，定时刷新
2. 会话缓存: 活跃会话保留30分钟
3. 意图缓存: 相似问题直接匹配历史意图
```

### 并发处理

```
1. 每个玩家独立会话
2. Skill 调用可并行（如同时检索多个知识点）
3. 飞书写入异步（不阻塞回复）
```

## 监控指标

```
1. 意图识别准确率
2. 平均对话轮数
3. Bug 收集完成率
4. 玩家满意度（可添加评分）
5. 平均响应时间
```
