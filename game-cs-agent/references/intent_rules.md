# 意图识别规则库

## 攻略查询意图 (guide_qa)

### 触发关键词

**获取类**：
- "怎么获得"、"如何获得"、"哪里获得"
- "怎么得到"、"如何得到"
- "在哪里找"、"去哪里找"
- "怎么获取"、"如何获取"

**位置类**：
- "在哪里"、"去哪"、"什么地方"
- "怎么到达"、"怎么去"

**方法类**：
- "怎么做"、"如何制作"
- "怎么解锁"、"怎么开启"
- "怎么升级"、"怎么提升"

**说明类**：
- "是什么"、"有什么用"
- "怎么用"、"如何使用"
- "什么意思"

### 示例匹配

| 玩家输入 | 意图 | 置信度 |
|---------|------|--------|
| "小麦怎么获得" | guide_qa | 0.95 |
| "商店在哪里" | guide_qa | 0.95 |
| "这个道具有什么用" | guide_qa | 0.90 |
| "等级怎么提升" | guide_qa | 0.90 |
| "第三章怎么解锁" | guide_qa | 0.90 |

## Bug 上报意图 (bug_intake)

### 触发关键词

**消失/缺失类**：
- "不见了"、"消失了"、"没了"
- "找不到"、"没有显示"
- "丢失了"、"不见了"

**功能异常类**：
- "点不动"、"点不了"、"没反应"
- "打不开"、"进不去"
- "无法"、"不能"、"不可以"

**显示异常类**：
- "黑屏"、"白屏"、"花屏"
- "错位"、"重叠"、"显示不全"
- "闪退"、"崩溃"、"卡住"

**数据异常类**：
- "少了"、"多了"、"不对"
- "回档了"、"重置了"
- "没收到"、"没到账"

**性能类**：
- "卡顿"、"很卡"、"延迟"
- "发热"、"耗电快"
- "加载慢"、"转圈圈"

### 示例匹配

| 玩家输入 | 意图 | 置信度 |
|---------|------|--------|
| "我的商店不见了" | bug_intake | 0.95 |
| "任务完成不了" | bug_intake | 0.90 |
| "游戏闪退" | bug_intake | 0.95 |
| "按钮点不动" | bug_intake | 0.90 |
| "物品消失了" | bug_intake | 0.95 |

## 模糊地带 (需澄清)

### 特征

**可能是攻略也可能是 Bug**：
- "XX 不工作" → 不知道怎么操作？还是操作了没反应？
- "XX 不生长" → 缺少条件？还是满足条件但不生长？
- "拿不到 XX" → 不知道方法？还是方法对了但没拿到？
- "XX 没反应" → 不知道怎么触发？还是触发后没效果？

### 澄清话术

| 玩家输入 | 澄清问题 |
|---------|---------|
| "我的作物不生长" | "是不知道怎么让作物生长，还是浇水了也不生长？" |
| "任务没反应" | "是不知道怎么接任务，还是接了任务没反应？" |
| "商店不工作" | "是找不到商店，还是商店点不开？" |
| "拿不到奖励" | "是不知道怎么领奖励，还是领了没到账？" |

### 澄清后的路由

```
玩家: "浇水了也不生长"
    ↓
意图: bug_intake (0.85)

玩家: "不知道怎么让作物生长"
    ↓
意图: guide_qa (0.90)
```

## 对话管理意图

**结束对话**：
- "谢谢"、"明白了"、"好的"
- "没事了"、"没问题了"
- "拜拜"、"再见"

**请求人工**：
- "找人工"、"找客服"
- "转人工"、"人工服务"

**取消/放弃**：
- "算了"、"不用了"、"取消"
- "不管了"、"就这样吧"

## 复合意图处理

**一句话多个意图**：
```
玩家: "小麦怎么获得，还有商店不见了"

处理:
1. 分句: ["小麦怎么获得", "商店不见了"]
2. 分别识别: [guide_qa, bug_intake]
3. 优先级排序: bug_intake > guide_qa
4. 先处理 Bug，再处理攻略
```

**上下文相关的复合意图**：
```
玩家: "小麦怎么获得"
Bot: "小麦可以通过种植..."
玩家: "还有玉米，另外我的商店不见了"

处理:
1. "还有玉米" → 延续 guide_qa 上下文
2. "商店不见了" → 新意图 bug_intake
3. 先回答玉米，再切换至 Bug 收集
```

## 置信度计算

```
confidence = base_score * keyword_match * context_bonus

base_score: 基础分 (0.5)
keyword_match: 关键词匹配度 (0-1)
context_bonus: 上下文加成 (1.0-1.2)

阈值:
- >= 0.8: 直接路由
- 0.5-0.8: 需澄清
- < 0.5: 无法识别
```
