# 活动类型框架

本文件供 `event_ops` skill 使用。与 AGENTS.md 中「活动运营类型与机制规律」对齐。

---

## 活动类型与机制

| 活动类型 | 机制 | 主要效果 | 常见副作用 |
|---------|------|---------|----------|
| **限时冲刺活动** | 短时间内完成任务获得奖励 | 拉升 DAU、制造 FOMO 感 | 活动强度过高导致玩家疲劳；奖励门槛过高导致参与率崩溃 |
| **养成长线活动** | 每日签到/积累打卡类 | 提升 7–30 日留存、增加日活粘性 | 一旦中断玩家会有挫败感，反而流失 |
| **社交裂变活动** | 邀请好友、组队任务 | 拉新、激活沉睡用户 | 社交压力过大导致用户反感；奖励结构设计不当变成刷号 |
| **消耗清库活动** | 使用积累的货币/道具换取奖励 | 控制通货膨胀、回收虚拟货币 | 玩家感知为「被迫消耗」；频繁出现会降低货币价值感 |
| **限定皮肤/角色活动** | 限时抽卡或限时购买 | 集中变现、制造稀缺性 | 错过的玩家产生被剥夺感；保底机制不完善引发强烈反弹 |
| **节假日主题活动** | 与现实节日绑定的特别内容 | 情感联结、短期留存峰值 | 节日内容质量低于预期会特别刺眼（对比强烈） |

---

## 活动节奏设计原则

- 大版本发布后应有「蜜月期」（活动密度适中，让玩家消化内容）。
- 版本末尾用小活动维持活跃。
- 版本空档期是留存风险最高的时段，需要有计划填充。
- 避免连续高强度限时冲刺导致疲劳。

---

## 方案输出要点（plan 模式）

- 活动类型与目标（DAU/留存/变现/拉新/消耗等）。
- 机制简述与规则要点。
- 节奏建议（时长、与版本/节假日的相对位置）。
- 风险提示（对应上表常见副作用及缓解思路）。

---

## 复盘输出要点（review 模式）

- 活动目标 vs 实际达成（指标对比）。
- 参与率、完成率、负面反馈归纳。
- 与同类活动或历史活动的对比（若有数据）。
- 下期建议（保留/调整/规避）。
