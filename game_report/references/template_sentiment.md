# 舆情分析报告模板（参考）

本模板作为**参考素材**，在撰写舆情分析类报告时可查阅。报告结构不由模板决定，而由 Phase 3.0 自我追问的「核心发现清单」决定。

**使用方式：**
- 先完成自我追问五层框架（表象→矛盾→归因→推演→机会）
- 根据追问结果决定需要哪些 section、什么顺序
- 不确定某 section 如何撰写时，参考本模板的写作方式

**通用撰写原则：**
- 开篇结论必须为单独一段 8–15 句，仅文字，不含表格或列表
- 每个一级 section 必须包含**至少 2–3 段连贯的深度解读段落**（每段 3–5 句）
- **禁止在报告中使用任何 emoji**；等级、程度、热度等一律用文字
- 除表格与要点外，每个 section 须包含**至少一段连贯的段落式深度解析**

## 开篇结论

本段为结论先行的高洞察摘要，必填。

**格式：** 一段连贯大段落（8–15 句或约 200–400 字）。结论先行：先给出本报告核心结论、关键发现与风险/机会判断，再以简要依据支撑。高洞察：回答本质问题、主要原因、若不干预如何演变；禁止仅复述数据或表格。**本段仅文字，不包含表格或列表。**

## 一、执行摘要

用一个表格呈现最核心的量化指标，让读者在5秒内了解游戏现状。

**格式：**

```
| 指标 | 本期数值 | 上期数值 | 变化趋势 | 预警等级 |
|------|:--------:|:--------:|:--------:|:--------:|
| **综合评分** | {X.X}/10 | {X.X}/10 | ↑/↓/→ {±X.X} | 正常/关注/警戒 |
| **评价总数** | {X,XXX} 条 | {X,XXX} 条 | ↑/↓ {XX%} | 正常/关注/警戒 |
| **负面评价占比** | {XX%} | {XX%} | ↑/↓ {±XX%} | 正常/关注/警戒 |
| **下载量** | {XXX} 万 | {XXX} 万 | ↑/↓ {XX%} | 正常/关注/警戒 |
| **关注数** | {XXX} 万 | {XXX} 万 | ↑/↓ {XX%} | 正常 |
| **当前版本** | {版本号}（{赛季/主题}） | - | - | - |
| **整体趋势** | {一句话判断，如"玩家反馈分化明显"/"整体口碑良好"} | - | - | - |
```

**本期解读：**（本小节须写 2–3 段连贯段落，每段 3–5 句；不得以要点替代段落。）
{综合上表数据，说明本期舆情的核心特征、与上期或行业基准的对比、以及最值得管理层关注的一两个信号。不得只复述表格数字。}

**填写指引：**
- 整体趋势判断标准：评分 ≥ 8.0 → "口碑优秀"；7.0-7.9 → "口碑中等偏上"；6.0-6.9 → "评价分化"；< 6.0 → "口碑偏差，争议较大"
- 预警等级标准（用文字，禁止 emoji）：
  - 正常：评分变化 < ±0.2，负面占比 < 30%
  - 关注：评分下降 0.2-0.5，负面占比 30-45%
  - 警戒：评分下降 > 0.5，负面占比 > 45%，下载量降幅 > 30%
- 如为首期报告，"上期数值"填"-"，"变化趋势"填"基准值"
- 如有明显近期趋势变化（如某次更新后评分骤降），在趋势一栏注明具体原因

```
### {维度2}
...
```

**本节小结：**
**正面评价解读：**（本小节须写 2–3 段连贯段落，每段 3–5 句；不得以要点替代段落。）
{概括本期正面舆情的整体格局——哪些维度是稳定优势、哪些是近期上升亮点、对留存或口碑的贡献分别是什么；可与负面 section 形成对照，指出"玩家认可但尚未被充分满足"的方向。}

- 每个维度的"**分析：**"段落为必填项，即使数据有限也需给出 1-2 句基于现有引用的推断性解读，不得省略

## 四点五、玩家分层分析

分析不同生命周期玩家的满意度差异，识别关键流失风险点。

**格式：**

```
| 玩家类型 | 占比估算 | 核心痛点 | 满意度 | 流失风险 | 关键诉求 |
|---------|:--------:|---------|:------:|:--------:|---------|
| **新手玩家**(0-7天) | ~{XX%} | {新手引导/上手难度/匹配体验} | 中 | 高/中/低 | {最希望改进的1-2项} |
| **成长期玩家**(7-30天) | ~{XX%} | {成长曲线/内容深度/养成压力} | 较高 | 中/低 | {最希望改进的1-2项} |
| **核心玩家**(>30天) | ~{XX%} | {内容重复/平衡性/竞技环境} | 中 | 中/低 | {最希望改进的1-2项} |
| **流失/观望玩家** | ~{XX%} | {回归门槛/版本脱节/失望累积} | 低 | 已流失 | {可能吸引回流的要素} |

**关键发现：**
{2-3句话说明不同玩家群体的差异化问题，如"新手玩家对匹配机制不满最为突出（占新手差评65%），而核心玩家更关注平衡性问题"}

**分层解读段落：**（本小节须写 2–3 段连贯段落，每段 3–5 句；不得以要点替代段落。）
{从"谁在流失、谁在观望、谁仍满意"的角度做连贯分析；说明各群体痛点之间的关联（如新手流失是否与核心玩家环境恶化有关）；指出当前数据下最需要优先挽留或激活的玩家群及理由。}

**数据来源说明：**
{说明如何从评论内容推断玩家类型，如"根据评论中提及的游戏天数、等级、赛季信息等判断"}
```

**填写指引：**
- 占比估算：根据评论中透露的游戏时长信息推断，如"刚玩3天"→新手玩家
- 满意度：根据该群体评价的正负面比例判断
- 流失风险：结合痛点严重程度和该群体评价趋势变化判断
- 关键诉求：提取该群体高频提及的改进建议

## 五、负面舆情与痛点

用排名表呈现主要痛点，按严重程度降序排列，并量化其商业影响。

**格式：**

```
| 排名 | 痛点问题 | 严重程度 | 影响范围 | 商业影响预估 | 典型反馈 |
|:---:|----------|:--------:|---------|-------------|----------|
| **1** | **{最严重痛点}** | 极高 | 影响{XX%}玩家<br>{X,XXX}条提及 | **预计导致{XX%}留存下降**<br>或{具体影响描述} | *"{典型差评引用}"*[N] |
| **2** | **{次严重痛点}** | 高 | 影响{XX%}玩家<br>{X,XXX}条提及 | {具体影响描述} | *"{典型差评引用}"*[N] |
| **3** | ... | 较高/中 | ... | ... | ... |
（列出前5-10个痛点）

**痛点成因与关联分析：**
**痛点成因分析：**（本小节须写 2–3 段连贯段落，每段 3–5 句；不得以要点替代段落。）
{分析排名靠前的痛点之间的因果或关联（如某技术问题是否放大商业化抱怨）；说明哪些问题属于"急性危机"、哪些属于"慢性积累"；对商业影响预估的依据做简要说明。}

**代表性差评：**
> "{完整差评引用1}"[N]

> "{完整差评引用2}"[N]
```

**填写指引：**
- 严重程度评估（用文字，禁止 emoji）：极高 = 大量重复提及、点赞极高、直接影响评分；低 = 少数玩家提及；中间为 高/较高/中
- 影响范围：根据提及频次占总评价量的比例 + 点赞数估算受影响玩家比例
- 商业影响预估：
  - 关键功能问题（匹配/闪退）→ 预计导致 15-30% 留存下降
  - 体验问题（平衡性/内容）→ 影响核心玩家满意度，可能流失 10-20% 核心用户
  - 商业化问题（付费压力）→ 影响新增转化，降低 LTV
  - 技术问题（卡顿/Bug）→ 影响 XX% 设备用户，降低评分
- 典型反馈列中用斜体引用，保持原文风格，可适当裁剪
- "代表性差评"选2-4条最能代表主要痛点的完整评论

**填写指引：**
- 核心结论要有数据支撑（引用评分、下载量、提及频次等）
- 优先级标准：
  - **P0**（紧急且重要）：直接影响留存/评分的核心问题，需立即处理
  - **P1**（重要不紧急）：影响用户体验，但短期内不会导致大量流失

- 参考案例：优先引用同品类头部游戏的成功案例，增加方案可信度
- 预期效果：尽量量化（如"提升XX%留存"、"减少XX%投诉"）
- 实施难度判断：
  - 低：仅需数值调整、UI优化等，1周内可完成
  - 中：需开发新功能，但技术难度不大，2-4周
  - 高：需重构核心系统，技术难度大或涉及多部门协作，>1月
- 责任方：明确具体部门，便于后续跟进
