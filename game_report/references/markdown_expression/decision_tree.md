# Markdown 表达决策树

**根据洞察类型选择最合适的表达格式**

## 快速决策表

| 洞察类型 | 推荐格式 | 使用场景 | 示例 |
|---------|---------|---------|------|
| **时间因果链** | 时间线 / 流程图 | 展示事件先后与因果关系 | "版本发布后口碑如何演变" |
| **多方对比** | 对比表格 | 本品vs竞品、现状vs历史、不同平台差异 | "TapTap vs Reddit 口碑差异" |
| **归因拆解** | 层级列表 / 树状结构 | 逐层拆解问题归属 | "付费争议归因：商业化层+运营层" |
| **情绪证据** | 引用块 + 来源标注 | 玩家原声、典型反馈 | "> '这价格太离谱了' [3]" |
| **反事实推演** | 折叠块对比 | "如果当时...现在会怎样" | "如果提前沟通定价策略..." |
| **矛盾/异常发现** | 高亮块 + 解释 | 强制注意力 | "评分8.5但情绪负面：刷分嫌疑" |
| **数据支撑细节** | 折叠表格 / 附录 | 不干扰主线 | "完整评分分布（点击展开）" |
| **复杂关系网络** | 矩阵图 / 交叉表格 | 多维度交叉分析 | "玩家分层 × 诉求类型矩阵" |

## 详细使用指南

### 1. 时间因果链

**何时使用：**
- 需要解释"为什么现在会这样"
- 展示事件发展的先后顺序
- 分析某个决策的长期影响

**Markdown 格式：**
```markdown
### 口碑演变时间线

**T-7天** — 版本预告发布
- 玩家期待值较高，讨论热度上升
- 主要关注点：新角色技能设计

**T-3天** — 定价公告发布
- 负面情绪开始出现，集中在定价策略
- 代表性反馈：> "这个价格认真的吗？" [5]

**T+0天** — 版本正式上线
- 评分开始下滑，负面评价占比上升至 42%
- 核心矛盾：内容质量认可 vs 定价不满

**T+3天（当前）** — 口碑持续发酵
- 论坛出现有组织抵制迹象
- 风险：若官方沉默，可能演变为信任危机
```

**关键技巧：**
- 每个时间点包含「发生了什么」+「玩家反应」+「数据支撑」
- 用引用块插入典型玩家反馈
- 最后总结趋势和风险

---

### 2. 多方对比

**何时使用：**
- 本品vs竞品分析
- 不同平台/区域差异
- 现状vs历史对比

**Markdown 格式：**
```markdown
### 本品与竞品口碑对比

| 维度 | 本品（心动小镇） | 竞品A（星布谷地） | 竞品B（动物森友国） |
|-----|-----------------|------------------|-------------------|
| **TapTap评分** | 7.8 | 8.2 | 8.5 |
| **近30天评价情绪** | 负面上升 | 稳定偏正 | 稳定 |
| **核心痛点** | 定价争议 | 内容更新慢 | 社交功能弱 |
| **玩家称赞点** | 画风、自由度 | 剧情、角色 | 休闲节奏 |
| **风险等级** | 高（定价争议发酵） | 中 | 低 |

**关键发现：**
本品评分低于竞品，但差距主要在「定价争议」而非内容质量。画风和自由度仍是核心优势，但商业化策略正在侵蚀这一优势。
```

**关键技巧：**
- 对比维度要服务于核心发现
- 最后一列加「关键差异」或「启示」
- 表格后必须有解读段落

### 3. 归因拆解

**何时使用：**
- 问题诊断
- 责任归属分析
- 改进方向建议

**Markdown 格式：**
```markdown
### 付费争议归因分析

**表面现象：** 新版本角色定价引发大规模负面反馈

**逐层归因：**

1. **直接触发点**（运营层）
   - 定价公告发布时间不当（版本上线前3天才公布）
   - 缺乏预期管理，玩家感到"被突袭"
   - 历史付费玩家未获得足够尊重（无老玩家优惠）

2. **设计根因**（商业化层）
   - 定价锚点设置失误（与历史角色价格对比过于突兀）
   - 价值感知不足（新角色强度与定价不匹配）
   - 保底机制不透明

3. **系统性问题**（策划层）
   - 角色强度膨胀曲线失控，新角色必须更强才能卖动
   - 付费与免费玩家体验差距拉大

**责任归属：** 运营层（40%）+ 商业化层（35%）+ 策划层（25%）

**改进路径：** ...
```

**关键技巧：**
- 从表象到本质逐层深入
- 每层给出具体证据
- 最后明确责任比例和改进路径

### 4. 情绪证据

**何时使用：**
- 需要增强说服力
- 展示玩家真实声音
- 说明问题的严重程度

**Markdown 格式：**
```markdown
### 玩家核心不满

**定价超出预期**
> "这价格比我想象中贵了一倍，感觉被当韭菜割了。" [12]
> 
> "不是不能花钱，但得让我觉得值啊。" [18]

**历史付费玩家感到被忽视**
> "充了这么多，新角色一点优惠都没有，寒心。" [25]
> 
> "老玩家不如狗系列。" [31]

**对官方信任的动摇**
> "以前觉得官方挺良心的，这次真的失望了。" [45]
> 
> "再观望观望，不行就退坑。" [52]

**分析：** 玩家不满已从"定价高"演变为"信任受损"，这是更严重的信号。
```

**关键技巧：**
- 按主题分组，不要随机罗列
- 每段引用后标注来源 [N]
- 引用后必须有解读，说明这些引用的共同指向

### 5. 反事实推演

**何时使用：**
- 评估决策得失
- 展示不同选择的可能结果
- 为未来决策提供参考

**Markdown 格式：**
```markdown
### 反事实推演：如果提前沟通定价策略

**实际发生的情况：**
- 定价公告发布时间：版本上线前3天
- 玩家反应：负面评价占比 42%，评分下滑 0.6
- 核心矛盾：预期管理失败

**假设情景：提前两周公布定价并解释逻辑**

| 维度 | 实际发生 | 假设情景 |
|-----|---------|---------|
| 玩家准备时间 | 3天 | 14天 |
| 预期管理 | 失败 | 有机会逐步引导 |
| 负面评价占比 | 42% | 预估 25-30% |
| 评分影响 | -0.6 | 预估 -0.2 或持平 |
| 信任损耗 | 高 | 中 |

**关键差异点：**
提前沟通的核心价值不在于改变定价本身，而在于：
1. 给玩家消化和接受的时间
2. 展示官方尊重玩家的态度
3. 有机会通过解释获得理解

**启示：** 对于可能引发争议的决策，时间是最低成本的风险缓释工具。
```

**关键技巧：**
- 明确对比「实际」vs「假设」
- 用表格展示关键差异
- 最后总结「启示」而非仅描述差异

### 6. 矛盾/异常发现

**何时使用：**
- 数据中出现反常信号
- 需要强制读者注意
- 解释"看似不合理"的现象

**Markdown 格式：**
```markdown
### 异常信号：评分与情绪的背离

**现象：**
- TapTap 评分：8.5（较高）
- 近7天评价情绪：负面占比 58%
- 论坛讨论热度：上升 120%

**矛盾点：**
评分高但情绪负面，且讨论热度异常上升。这表明：

1. **可能原因A：有组织刷分**
   - 证据：新注册账号集中给出5星好评
   - 验证：查看好评账号注册时间和历史评价

2. **可能原因B：沉默的大多数 vs 发声的少数**
   - 证据：评分分布两极分化（5星和1星占比高）
   - 验证：分析评分分布曲线

3. **可能原因C：时间差效应**
   - 证据：负面评价集中在近3天，历史评分拉高了整体
   - 验证：查看评分趋势变化

**初步判断：** 原因B可能性最高，即核心玩家群体不满但泛用户群体尚未感知。
**风险：** 若核心玩家流失，评分将在未来1-2周内快速下滑。
```

**关键技巧：**
- 先明确展示「矛盾现象」
- 列出多个可能解释
- 给出「初步判断」和「验证建议」
- 最后说明「风险」

### 7. 数据支撑细节

**何时使用：**
- 详细数据会干扰主线阅读
- 读者可能想深入了解
- 需要展示数据来源的完整性

**Markdown 格式：**
```markdown
### 评分分布详情

**核心发现：** 评分两极分化，5星和1星占比显著高于中间评分。

<details>
<summary>点击查看完整评分分布</summary>

| 评分 | 数量 | 占比 | 近7天变化 |
|-----|------|-----|----------|
| 5星 | 1,245 | 35% | -5% |
| 4星 | 423 | 12% | -2% |
| 3星 | 312 | 9% | +1% |
| 2星 | 298 | 8% | +3% |
| 1星 | 1,287 | 36% | +12% |

**近7天评分趋势：**
- 平均评分：7.2 → 6.8
- 负面评价增速：日均 +15%
- 新评价中1星占比：42%

</details>

**解读：** 展开数据显示，负面趋势正在加速，1星评价增速远超其他评分。
```

**关键技巧：**
- 用 `<details>` 标签折叠
- 摘要中说明核心发现
- 折叠内容后必须有「解读」段落

### 8. 复杂关系网络

**何时使用：**
- 多维度交叉分析
- 展示群体差异
- 识别被忽视的细分问题

**Markdown 格式：**
```markdown
### 玩家分层 × 核心诉求矩阵

| 玩家类型 | 内容深度 | 社交体验 | 商业化 | 技术稳定 |
|---------|---------|---------|--------|---------|
| **硬核玩家** (20%) | 要求极高 | 中等 | 敏感 | 高 |
| **休闲玩家** (45%) | 中等 | 高 | 低敏感 | 中等 |
| **社交玩家** (25%) | 低 | 极高 | 低敏感 | 中等 |
| **付费玩家** (10%) | 高 | 中等 | 极高敏感 | 高 |

**关键发现：**
- **付费玩家**（10%）对商业化敏感度极高，是当前争议的核心发声群体
- **休闲玩家**（45%）对商业化不敏感，但占比最大，是口碑基本盘
- **风险：** 若付费玩家大规模流失，硬核内容产出将受影响，进而波及休闲玩家

**策略启示：** 需要分层沟通策略，不能一刀切回应。
```

**关键技巧：**
- 行和列的选择要服务于核心发现
- 单元格内容简洁（高/中/低或具体描述）
- 矩阵后必须有「关键发现」和「策略启示」

## 格式选择检查清单

在确定使用某种格式前，问自己：

- [ ] 这个格式能让洞察更易理解吗？
- [ ] 读者能否在30秒内抓住重点？
- [ ] 格式是否服务于内容，而非为了美观？
- [ ] 关键数据是否有引用标注 [N]？
- [ ] 表格/图表后是否有解读段落？
- [ ] 信息密度是否适中（不过载、不空洞）？

## 常见错误

| 错误 | 示例 | 修正 |
|-----|------|------|
| 表格后无解读 | 罗列数据后直接进入下一节 | 每个表格后必须跟2-3句解读 |
| 连续多个表格 | 三个表格连在一起 | 表格间插入文字段落，控制阅读节奏 |
| 引用无标注 | 直接引用玩家原话 | 必须标注来源 [N] |
| 折叠块无摘要 | `<details>` 无 summary | 必须写 summary，说明折叠内容的核心发现 |
| 为了格式而格式 | 明明可以用文字说明，硬要画表格 | 简单洞察用文字，复杂关系用表格 |
