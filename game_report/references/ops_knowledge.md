# 游戏运营分析参考库

本文件供 `game_report` skill 的 Phase 3 运营上下文评估步骤使用。在将数据转化为报告叙述之前，先查阅本文件完成运营阶段识别和问题归因。

---

## 一、运营阶段识别

根据以下信号判断游戏当前处于哪个运营阶段，这会影响所有问题的严重程度解读和建议优先级。

### 阶段信号对照表

| 运营阶段 | 数据信号 | 舆情信号 | 典型时间特征 |
|---------|---------|---------|------------|
| **买量冲刺期** | 下载量/安装量突然大幅增加；新用户评价大量涌入；整体评分波动加大 | 大量"刚下载""刚玩了XX天"的评价；好坏评价极端分化 | 通常在大版本发布前后、节假日前、联运合作期 |
| **版本发布期** | 评价数量激增；更新日志活动频繁；论坛新帖大量出现 | 大量"新版本好评/差评"；玩家集中讨论新内容 | 版本发布后 1-7 天 |
| **内容消化期** | DAU/评价量趋于稳定；论坛讨论逐渐深入（从"什么是X"到"X有什么问题"） | 玩家开始分析新版本的深层问题；攻略帖增多 | 版本发布后 1-3 周 |
| **版本末尾疲劳期** | 评价量下降；负面评价中"没什么可玩了""内容耗尽"频繁出现；论坛热度下降 | 玩家开始询问"下个版本什么时候来"；流失用户差评集中 | 距上次大版本更新 3-6 周以上 |
| **版本空档期** | 长时间无新内容；活跃度指标持续下滑；评论中"停更了吗"类问题出现 | 集中抱怨内容荒；对官方沉默感到不满；竞品讨论帖增加 | 超过同品类正常更新间隔的 1.5 倍以上 |
| **危机响应期** | 某一事件后（崩服/争议公告/抽卡风波）评分急剧下滑；差评量突增 | 集中、情绪化的负面舆情；社交媒体扩散迹象；论坛出现组织性讨论 | 触发事件后 24-72 小时为高峰期 |

### 阶段判断方法

1. **查看评价时间分布：** 如果最近 24-48 小时评价量是平时的 3 倍以上，通常是版本发布期或危机响应期
2. **检查论坛帖子主题：** "新内容求助"→内容消化期；"等更新等了好久"→版本末尾/空档期；"官方解释一下"→危机期
3. **看版本日志时间戳：** 最后一次更新到现在多少天，对照品类正常更新频率判断是否进入空档期
4. **检查竞品提及频率：** 评论中提及竞品的比例升高，通常是版本末尾疲劳期或危机期的信号

## 二、问题归因决策树

收集到负面舆情后，按以下流程判断问题根因，避免把运营问题当产品问题、把技术问题当策划问题来解决。

### 第一步：识别问题现象类型

```
玩家抱怨的是什么？

A. 游戏崩溃/卡顿/无法进入/Bug → 跳到【技术层归因】
B. 抽卡/付费/定价/"太贵了"/"必须充钱" → 跳到【商业化层归因】
C. 官方态度/沉默/补偿/承诺未兑现/活动设计太肝 → 跳到【运营层归因】
D. 玩法无聊/平衡太差/内容太少/系统设计烂 → 跳到【策划层归因】
E. 以上多个同时存在 → 多层归因，分别处理
```

### 技术层归因

**确认信号：**
- 特定设备/系统版本集中反馈
- 更新后新出现（更新前没有此类反馈）
- 描述具体（"三星S22更新后必崩"、"开团战必卡"）

**常见根因：**
- 版本热修复不完整
- 特定设备兼容性未测试
- 服务器承载能力不足（通常在活动/新版本开始时爆发）
- 第三方SDK更新导致冲突

**归因结论格式：**
`技术实现问题 — [具体表现] — 影响[X]类设备用户 — 建议[热修复/优化] — 责任方：技术组`

### 商业化层归因

**确认信号：**
- 集中在付费相关评价（星级评分1-2星 + 提及"钱"/"充值"/"抽卡"）
- 与版本内容质量投诉分离（内容评价正面，但付费评价负面）
- 出现"以前不是这样"（说明有历史对比感知）

**常见根因：**

| 现象 | 根因 |
|------|------|
| "太贵了" / "性价比差" | 定价超出目标用户支付意愿上限 |
| "必须充钱" / "不充玩不了" | 付费墙阻断核心体验，强迫感强 |
| "以前买的东西贬值了" | 历史付费资产保护不到位 |
| "概率太低" / "保底太远" | 抽卡机制设计不透明或保底距离不合理 |
| "活动奖励只有氪佬能拿到" | 付费与免费玩家的奖励层级差距过大 |

**归因结论格式：**
`商业化设计问题 — [具体现象] — [根因定位] — 建议[调整定价/增加保底/保护历史资产] — 责任方：商业化组`

### 运营层归因

**确认信号：**
- 投诉的是官方行为，而不是游戏本身（"官方不回应"、"说好的X没做到"）
- 活动相关投诉（"这活动太肝"、"时间太赶"、"奖励不值"）
- 沟通问题（"官方态度傲慢"、"公告写得不清楚"）

**常见根因：**

| 现象 | 根因 |
|------|------|
| "官方沉默/不回应" | 危机响应机制缺失或响应太慢 |
| "承诺的内容没做到" | 玩家预期管理失败，宣传与实际不符 |
| "活动太肝/强迫感强" | 活动设计的门槛设置脱离正常玩家行为 |
| "更新太慢/等好久了" | 版本更新节奏低于同品类预期，或公告不透明 |
| "玩家建议从不被采纳" | 官方与玩家社区的互动机制缺失 |

**归因结论格式：**
`运营决策问题 — [具体现象] — [根因定位] — 建议[调整响应策略/改进活动设计/加强玩家沟通] — 责任方：运营组`

### 策划层归因

**确认信号：**
- 对游戏设计本身的批评（"玩法就是有问题"）
- 与版本更新高度相关但不是Bug（更新前后体验质量变化）
- 长期积累的问题，不是单次事件触发

**常见根因：**

| 现象 | 根因 |
|------|------|
| "游戏越来越无聊" | 核心循环深度不足，内容消耗速率>更新速率 |
| "平衡太差，某XXX太强" | 平衡性调整机制或节奏有问题 |
| "新手不友好" | 新手引导设计缺陷 |
| "上手门槛太高/太低" | 目标用户定位与实际设计不匹配 |
| "游戏方向越来越不对" | 迭代方向与核心玩家预期产生偏离 |

**归因结论格式：**
`策划设计问题 — [具体现象] — [根因定位] — 建议[调整设计方向/平衡修复/内容补充] — 责任方：策划组`

## 三、时间维度风险评估

对每个识别出的核心问题，评估其在下版本的风险演变趋势。

### 风险升级判断标准

| 当前状态 | 下版本条件 | 风险升级程度 |
|---------|----------|------------|
| 技术Bug已有较多投诉 | 下版本没有专项修复公告 | 高风险升级：投诉量将持续积累 |
| 商业化差评增加 | 下版本有新付费内容上线 | 高风险：玩家敏感度已升高，新内容会被放大审视 |
| 内容耗尽投诉出现 | 下版本更新间隔超过正常周期 | 高风险：流失加速 |
| 平衡性投诉 | 下版本有新英雄/装备上线但无平衡修复 | 中风险：新内容可能加剧失衡感知 |
| 新手体验差评 | 下版本有大规模买量计划 | 高风险：新用户大量进入但无法留住 |
| 官方沟通差评 | 下版本有重大内容更新 | 低风险：好内容通常可以覆盖沟通问题 |

### 时间预警输出格式

```
时间维度预警

当前问题：[问题描述]
当前严重程度：高/中/低
下版本风险评估：[升级/维持/缓解]
依据：[1-2句说明为什么会升级或缓解]
建议处理时限：[立即/本版本内/下版本前]
```

## 四、运营阶段对分析的影响

不同运营阶段下，相同的数据信号意义不同。

| 数据现象 | 版本发布期的解读 | 版本末尾疲劳期的解读 |
|---------|--------------|------------------|
| 负面评价增加 20% | 通常是新内容问题反馈，可快速定位并修复 | 内容耗尽的慢性流失信号，问题更深层 |
| 评分下降 0.3 分 | 新版本中存在明确的体验痛点 | 长期问题积累的临界点，可能开始加速下滑 |
| 论坛活跃度下降 | 玩家在游戏而不是在论坛（正常） | 玩家离开游戏，失去讨论动力（警告） |
| 竞品提及增加 | 玩家在对比新内容质量 | 玩家已在迁移或评估迁移 |
