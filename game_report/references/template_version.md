# 版本内容分析报告模板（参考）

本模板作为**参考素材**，在撰写版本分析类报告时可查阅。报告结构不由模板决定，而由 Phase 3.0 自我追问的「核心发现清单」决定。

**使用方式：**
- 先完成自我追问五层框架（表象→矛盾→归因→推演→机会）
- 根据追问结果决定需要哪些 section、什么顺序
- 不确定某 section 如何撰写时，参考本模板的写作方式

**通用撰写原则：**
- 开篇结论必须为单独一段 8–15 句，仅文字，不含表格或列表
- 每个一级 section 必须包含**至少 2–3 段连贯的深度解读段落**（每段 3–5 句）
- **禁止在报告中使用任何 emoji**；等级、程度、热度等一律用文字
- 除表格与要点外，每个 section 须包含**至少一段连贯的段落式深度解析**

## 开篇结论

本段为结论先行的高洞察摘要，必填。

**格式：** 一段连贯大段落（8–15 句或约 200–400 字）。结论先行：先给出本版本核心结论、关键发现与风险/机会判断，再以简要依据支撑。高洞察：回答本质问题、主要原因、若不干预如何演变；禁止仅复述数据或表格。**本段仅文字，不包含表格或列表。**

## 一、版本概览

用一张表格快速呈现版本的核心信息，让读者在10秒内了解本次更新的规模与重点。

**格式：**

```
| 项目 | 本版本 | 上版本对比 |
|------|--------|-----------|
| **游戏名称** | {游戏名} | - |
| **版本号** | {X.X.X} | {上一版本号} |
| **版本主题** | {赛季名/版本主题，如无则填"常规更新"} | {上版本主题} |
| **发布日期** | {YYYY年M月D日} | {上版本日期} |
| **更新间隔** | - | {距上次更新X天} |
| **更新规模** | {大版本/中版本/小版本/热修复} | {对比：更大/相当/更小} |
| **核心亮点** | {一句话概括本版本最重要的1-2项更新} | - |

### 版本说明原文摘要
{从更新日志数据（TapTap / 官网 Patch Notes / YouTube 视频 transcript）中提取的版本说明首段，控制在100字以内}

### 更新节奏评估
{1-2句话评价本次更新节奏，如"本次更新间隔35天，符合官方月度大版本节奏"或"距上次更新仅7天，为紧急热修复"}
```

**版本概览解读：**（本小节须写 2–3 段连贯段落，每段 3–5 句；不得以要点替代段落。）
{综合上表与版本说明摘要，说明本版本在产品节奏中的定位——是大版本后的消化期补丁还是新赛季前的预热；与上版本相比规模与重点的变化；对玩家预期管理的影响。}

**填写指引：**
- 更新规模判断：新英雄/新赛季/核心玩法改动 → 大版本；数个系统优化+平衡 → 中版本；仅Bug修复/小调整 → 小版本/热修复
- 核心亮点聚焦在玩家最关心的内容（新英雄、重大机制改动等）
- 更新节奏评估：对比同品类游戏更新频率，判断是否过快/过慢

## 二、核心更新内容

按更新类别整理本次版本的全部内容，确保完整覆盖。

**格式：**

### 新赛季与活动
| 内容 | 详情 |
|------|------|
| {赛季名称} | {1-2句描述赛季主题与主要玩法变化} |
| {限时活动} | {活动名称与主要奖励} |
（如无，注明"本版本无新赛季"）

### 系统与功能优化
| 优化项目 | 变更内容 |
|----------|----------|
| {系统名称} | {具体改动描述} |
（列出本次更新的全部系统改动，不遗漏）

### 平衡性调整
| 对象 | 调整方向 | 具体变更 |
|------|----------|----------|
| {英雄/装备名} | 增强 ↑ / 削弱 ↓ / 重做 ⟳ | {具体数值或描述} |
（如本版本无平衡调整，注明"本版本无平衡性调整"）
```

**核心更新解读：**（本小节须写 2–3 段连贯段落，每段 3–5 句；不得以要点替代段落。）
{概括本版本更新的整体取向——偏内容扩充、偏平衡修正还是偏系统优化；哪些改动对多数玩家可见、哪些主要影响核心/竞技玩家；与官方或社区此前承诺的呼应或落差。}

**填写指引：**
- 玩家关注度基于：是否是热门角色/机制、更新前社区讨论热度、是否为玩家呼声已久的改动
- 系统改动要覆盖所有类别，包括 UI、匹配、社交、商城等
- 平衡性调整尽量列出具体数值（如有）

## 三、平衡性变更详情

对平衡调整进行深度分析，评估其对游戏生态的影响。若本版本无平衡调整，本 section 可简化为"本版本无平衡性调整"。

**格式：**

```
### 强化对象分析
{英雄/装备名1}（{调整前状态} → {调整后状态}）
- **调整内容：** {具体改动}
- **影响预估：** {对游戏平衡/对应位置强度的影响判断}

### 削弱对象分析
{英雄/装备名}（{调整内容}）
- **调整内容：** {具体改动}
- **影响预估：** {是否会导致角色跌出强度榜？影响玩家使用率？}

### 本版本平衡总结
（本小节须写 2–3 段连贯段落，每段 3–5 句；不得以要点替代段落。）
{总结本版本平衡调整的整体方向与设计意图；说明强化/削弱对象之间的逻辑关系（如是否形成"此消彼长"）；对天梯或赛事环境可能产生的影响做简要预判。若本版本无平衡调整，本段可改为"本版本无平衡性调整，上述分析不适用"。}
```

**填写指引：**
- 本 section 内"强化对象分析"、"削弱对象分析"各条中的"影响预估"须以连贯 1-2 句段落书写，不得仅用短语

## 四、玩家对本次更新的反应

收集玩家在各平台（TapTap 评价页、Reddit 论坛、YouTube 评论区等）对本次更新的真实反应，并按玩家群体分层分析。

**代表性好评：**
> "{玩家原话引用}"[N]

|----------|-----------|:--------:|---------|-------------|
| {争议内容} | {争议概括} | 高/中/低 | 影响{XX%}玩家 | 老玩家/付费玩家/特定位置玩家 |

**代表性争议评论：**
> "{玩家原话引用}"[N]

{2-3句话总结玩家对本次更新的整体态度，如"本次更新整体获得正面评价，新英雄和赛季内容受到好评，但匹配机制调整引发核心玩家较大争议，新手玩家则反应良好"}
```

**玩家反应深度解读：**（本小节须写 2–3 段连贯段落，每段 3–5 句；不得以要点替代段落。）
{分析正面与争议/负面反应背后的成因——是期望管理问题、沟通不足还是设计本身引发分歧；不同玩家群体反应差异与各群体诉求的对应关系；本次更新对口碑或留存的短期影响预判。}

**填写指引：**
- 热度和严重程度基于点赞数和提及频次
- 影响范围：根据抱怨提及频次占比估算
- 主要受众/抱怨群体：根据评论中透露的玩家身份判断（如"我才玩3天"→新手，"S1老玩家"→核心玩家）
- 争议内容重点关注：期望落空（"这什么更新"）、平衡调整不满、Bug 多等
- 不同玩家群体满意度：根据该群体评价的正负面比例判断
