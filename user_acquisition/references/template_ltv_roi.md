# LTV / 买量 ROI 测算报告模板

本模板用于 `user_acquisition` skill 的输出。

---

## 报告头部

```
# LTV / 买量 ROI 测算说明

**编制日期：** {YYYY年M月D日}
**产品/游戏：** {若有}
**数据来源：** {用户提供的文件/表格说明}
**测算焦点：** {LTV / 买量ROI / 回收周期 / 综合}

---
```

---

## 一、数据概况

- **数据范围**：时间窗口、渠道/维度、主要列名。
- **口径说明**：ARPU/付费率/LTV/CPI 等在本数据中的定义（若有歧义则注明）。

---

## 二、测算结果（按 focus 选填）

### LTV

- **公式与假设**：采用的公式（如 LTV = ARPU × 留存月数）、假设（如平均留存月数 = X）。
- **结果**：LTV 值或区间、单位（元/美元等）。
- **解读注意**：样本、时间窗口、品类对比。

### 买量 ROI

- **公式**：如 ROI = LTV / CAC 或 (LTV × 付费率) / CAC。
- **假设**：CAC 口径、LTV 时间窗口（如 30 日）。
- **结果**：ROI 值或区间。
- **解读注意**：回收是否覆盖成本、与行业基线对比（若有）。

### 回收周期

- **逻辑**：达到 CAC 所需累计收入对应的天数（按留存/付费曲线或简化假设）。
- **结果**：N 日回本或区间。
- **解读注意**：假设敏感性、渠道差异。

---

## 三、建议与局限

- **可执行建议**：如优化投放、分渠道看 ROI、延长观察窗口等。
- **局限**：数据局限、假设局限、不可替代专业 BI 等。
