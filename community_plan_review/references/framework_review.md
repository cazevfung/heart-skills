# 社区复盘分析框架（产品无关）

本框架用于 `community_plan_review` skill 的 **mode=review**。定义复盘报告应覆盖的指标分类、分析维度与健康判断方式；不写死具体数值，以「相对目标/基线」描述。

**与 OKR 对齐**：若用户提供 **OKR 文档**（O + KR + 控制指标），复盘中指标应尽量与 **KR/控制指标** 一一对应（如 KR1→内容表现，KR2→用户增长，KR3→口碑），并在报告中注明「目标来源：某 OKR 文档」或等价表述；具体 O/KR 文案由用户文档带入。

---

## 一、指标分类

复盘报告需覆盖以下三类指标，每类下列项为常用子项；按产品实际有的指标填写，缺失则注明「数据不足」。

| 类别 | 子指标 | 说明 |
|------|--------|------|
| 内容表现 | 总播放量、分平台播放量 | 可区分自然 vs 加热 |
| 内容表现 | 完播率（整体/分平台/分内容类型） | 短视频可增 5s 完播率、2s 跳出率 |
| 内容表现 | 互动率（点赞、评论、转发） | 可按平台、按内容类型对比 |
| 用户增长 | 粉丝增量、总量 | 分平台 |
| 用户增长 | 活跃度（可选） | 如主页访问、私域入群等 |
| 用户增长 | 私域运营（可选） | QQ 群、Kook 等规模与活跃 |
| 口碑 | 各商店/渠道评分 | 应用商店、TapTap 等 |

**扩展**：新增指标时在对应类别下追加一行。

### 常见 OKR 指标类型（供对照）

若复盘对照 OKR，下列类型常出现在 KR 或控制指标中；具体目标值由用户文档带入，此处仅作类型提示。

| 类别 | 常见 KR/控制指标类型 | 说明 |
|------|----------------------|------|
| 内容 | 总播放量、分平台分配（如 B:抖:快 比例） | 可带自然 vs 加热 |
| 内容 | 完播率（长视频/短视频区分阈值） | 如 10 分钟+ 某%、3～5 分钟某%、5s 完播某% |
| 内容 | 单条目标（如某节点前瞻视频观看量） | 大节点重点内容 |
| 用户 | 粉丝总量、分平台占比或目标 | 各平台粉丝数及占比 |
| 用户 | 私域规模（如 QQ 群人数） | 可选 |
| 用户 | 活跃（7 日/30 日动态平均活跃等） | 可选 |
| 口碑 | 各商店平均评分、单渠道要求（如 Tap>X、Steam 多半好评） | 不写死具体分数 |
| 口碑 | 评价数、推荐率（如 WeGame 推荐率） | 可选 |
| 内容（海外/自然流侧重） | 自然观看 organic views、自然曝光 organic impressions | 若 OKR 强调「自然」则单独列出与加热区分 |

## 二、分析维度

复盘正文中建议按以下维度组织分析与结论：

1. **按内容类型对比**：不同内容类型（娱乐短视频、教学攻略、深度内容等）的播放、完播、互动表现；识别最优/最弱类型。
2. **按平台对比**：各平台流量占比、完播率差异、互动与转化效率差异；归纳平台特性（如哪家自然流为主、哪家加热效率高）。
3. **爆款/典型内容总结**：列举表现突出的单条或系列，说明其形式、主题、数据亮点，便于复用经验。
4. **平台差异结论**：用 1～3 句话概括各平台定位与效果差异（如「A 站为流量主阵地；B 站完播与转化更优」）。

5. **用户阶段 × 内容（可选）**：若方案或 OKR 中有「认知/考虑/转化」三阶段，复盘中可按各阶段对应内容类型做效果对比——认知阶段：热点类、抽象主题、卡点混剪等；考虑阶段：官宣定档、实机 PV、商业化 PV、深度开发者、CG 等；转化阶段：详细玩法介绍等。便于与内容运营方针对齐。

6. **有效内容 vs 需调整内容**：复盘中建议分两块归纳——**什么是有效的**（数据或反馈验证过的内容形态、排期、视觉风格等）；**什么是需要调整的**（混剪效果一般、排期过紧、管线限制等），并附可重复验证的改进方向（如「排期提前」「色彩/风格统一」待下次验证）。

---

## 三、健康判断指引

- **达成/预警**：对照产品既定 OKR 或目标（如「总播放 X 万」「自然流占比 > Y%」），达成则标注达成，未达成则标注预警并简述差距；若用户未提供目标，可写「相对基线」或「行业常见区间」的定性判断。
- **避免写死数值**：框架内不规定「完播率多少算健康」，由复盘时根据用户提供的目标或文档中的基线填写；若无数值，可仅做趋势与对比描述。

与 game_report 的「执行摘要 + 分节表格」风格对齐，便于与现有报告体系一致。

---

## 四、复盘洞察类型（可选）

书写「可复用结论与建议」时，可参考下列**洞察类型**（不写死具体产品），便于从复盘中提炼可迁移经验：

| 类型 | 说明 | 示例方向 |
|------|------|----------|
| 大物料/混剪效果 | 动画与实机画面差距、风格统一度对品牌混剪效果的影响 | 混剪效果一般时归因：画面风格是否统一 |
| 角色/人物向内容 | 色彩、人设强度、排期早晚对自然观看或完播的影响 | 色彩更亮丽、排期更早可能更易积累兴趣；可标注「待下次验证」 |
| 排期与管线 | 制作管线导致的排期紧张、释出时机与数据关系 | 排期提前 vs 排期紧的对比；管线所限时的改进方向 |
| 内容规划反馈维度 | 系统玩法向、IP 内容向、英雄向、运营向、二创激励等 | 复盘或方案迭代时可按这些维度写「可优化栏目」与参考示例 |
