# 社区方案报告模板

本模板用于 `community_plan_review` skill 的 **mode=plan**。按以下 section 顺序生成报告；若用户提供文档则从文档抽取填入，否则标注「待补充」。

---

## 报告头部（不作为 section 编号）

```
# {产品/游戏名称} 社区建设方案

**报告日期：** {YYYY年M月D日}
**方案范围：** {如：公测前社区建设 / 上线期多平台运营}
**核心节点：** {如：公测日、上线日}

---
```

---

## 一、背景与核心挑战

简述产品所处阶段与社区建设面临的双重或多重挑战（如内容品类多样化+品质专业化+产能规模化；或多渠道基建薄弱需短期建立多平台运营体系）。1～3 段即可。

**填写指引：** 若用户提供文档则从文档「背景」节抽取；否则按产品与节点自行概括并标注「待补充」可细化处。

---

## 二、总体策略与阶段规划

按 [framework_community_plan.md](framework_community_plan.md) 的阶段模型填写下表。

| 阶段 | 时间 | 目标 | 产出类型/重点 |
|------|------|------|---------------|
| 阶段一 | {起止日期} | {目标简述} | {内容类型或数量预期} |
| 阶段二 | {起止日期} | {目标简述} | {内容类型或数量预期} |
| 阶段三 | {起止日期} | {目标简述} | {内容类型或数量预期} |

**填写指引：** 与 framework 中阶段模型对应；若采用测试期（CBT）/公测期（OBT）等阶段类型，可在目标列注明阶段类型及侧重（教会/服务）；时间、目标可从用户文档解析或标注待补充。

### 3.3 表达内容与排期原则（可选）

**表达内容**：若方案按「游戏」「角色」「IP/世界观」拆块，可简述各块在本阶段的侧重（游戏：版本/玩法/攻略；角色：背景/关系/派系/设计；IP：历史/人文/地貌等）。

**排期原则**：围绕节点（如版本更新、英雄上线）反推；总条数或区间（含 PGC+UGC）；每周 PGC 条数；释出集中日（如周四～周六）。

**填写指引：** 与 framework_community_plan 的「内容矩阵维度」「平台策略指引」「内容排期结构」「攻略分发与游戏内导流」对应；若用户提供文档则从文档表格抽取。

## 四之二、KOC / PUGC 规划（可选）

若方案包含 KOC 或 PUGC，可简述：目标（高质量内容 + 声量基本盘）；签约与锁定（竞品 KOC、自培养头部、目标人数/平台）；激励与支持（梯度奖励、流量/CDkey/推流、物料与版本前瞻）；自培养与全民 UGC（奖励标准、素材支持、征集活动）；供应商/MCN（可选）。详见 framework「KOC/PUGC 规划」。

---

## 五、目标与 OKR（可选）

若用户提供 OKR 文档，在此列出 O 及各 KR、关键控制指标（仅列类型与目标值槽位，如「KR1：PGC 播放量—总目标及分平台」），便于后续复盘对照。

| O / KR | 描述 | 关键控制指标 |
|--------|------|--------------|
| O | {如：提升整体口碑及行业地位} | - |
| KR1 | {如：PGC 内容 N 天总播放量} | {分平台分配、完播率要求等} |
| KR2 | {如：各平台粉丝总量} | {分平台目标、私域、活跃等} |
| KR3 | {如：各商店平均评分} | {各渠道要求、评价数等} |

若无 OKR 可写「待补充」或「见单独 OKR 文档」。
