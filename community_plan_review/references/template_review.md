# 社区复盘报告模板

本模板用于 `community_plan_review` skill 的 **mode=review**。按以下 section 顺序生成报告；指标分类与 framework_review 一一对应。若某项数据无法获取，注明「数据不足」或「待补充」，不跳过该 section。

---

## 报告头部（不作为 section 编号）

```
# {产品/游戏名称} 社区运营复盘报告

**报告日期：** {YYYY年M月D日}
**复盘时段：** {起止日期或如「公测后 30 天」}
**数据来源：** {用户提供的文档/平台列表/说明}
**目标来源（可选）：** 若对照某 OKR 文档，此处写文档名或如「公测后 30 天 OKR」

---
```

## 五、总结与建议

### 5.1 平台差异化效果总结

1～3 句话概括各平台定位与效果差异（如流量主阵地、完播/转化最优平台等）。

### 5.2 OKR 达成概览（可选）

若本次复盘对照某 OKR 文档，用 1 段或表格汇总各 KR/控制指标达成情况（达成/未达成、差距简述）。

### 5.3 可复用结论与建议

- **内容侧**：{如：福利与实用信息表现突出；某类内容完播/互动最佳}
- **平台侧**：{如：某平台适合做 X，某平台加热效率高/低}
- **后续建议**：{1～3 条可执行建议}

**复盘洞察类型（可选）**：书写时可参考 framework_review「复盘洞察类型」——大物料/混剪效果、角色向内容与排期、排期与管线、内容规划反馈维度（系统玩法向、IP 内容向、英雄向、运营向、二创激励）等，便于提炼可迁移经验。

**填写指引：** 与 framework_review「分析维度」中的爆款总结、平台差异结论对应；保持产品无关表述以便复用到其他项目。
